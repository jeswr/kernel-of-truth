**Status:** adversarial design review, 2026-07-07. Bead: `kernel-of-truth-tt5`.
**Commission (maintainer, verbatim intent):** "critically review the design of the kernel. I note that it likely has a lot of my own personal biases from working with semantic web technologies." The brief: stress-test the design from perspectives that did *not* grow up on RDF.

Each objection carries (a) the steelman, (b) what the current design already answers, with citations, (c) a verdict — **DESIGN CHANGE RECOMMENDED** / **RESIDUAL RISK ACCEPTED** / **REFUTED** — and, for design changes, the minimal concrete change.

## Lens 1 — the anti-semantic-web systems designer

The question for every inherited structure: is it load-bearing for the LLM-kernel goal, or is it Solid/linked-data muscle memory?

### 1.1 Two serialisations is a smell, and the RDF one is off the critical path

**Steelman.** The kernel's identity layer hashes an RDF record form: bnode trees under a closed SHACL shape, RDF lists for ordering, RDFC-1.0 canonicalisation, a duplicate-first-degree-hash gate, dataset-poisoning defences, SCC component datasets (gist §5–§6). Meanwhile *everything the programme has actually built and measured* — the encoder, kernel-v0, math-v0, physics-v0, molecules-v0, Phase X, E2 — runs on JSON ASTs (`kot-ast/1`, `pm-ast/1`, `kot-phys/1`) with **placeholder URNs**, real minting deferred (data/kernel-v0/README.md: "minting real `urn:concept:` identities is the concept-hash pipeline's job, out of scope here"; bead `kernel-of-truth-arq` still open). The math design even names the pattern: "`pm-ast/1` — the authoring mirror of an eventual RDF record form, exactly as `kot-ast/1` mirrors profile-1's RDF form" (design-math-sector.md §3.1). So the system maintains two isomorphic record languages, and the one carrying identity is the one with no consumer in the experiment pipeline.

What the RDF choice *costs*, concretely:

1. **Ordering fights the data model.** RDF is an unordered triple set; the grammar is ordered everywhere (clause lists, referent lists, exponent lists, argument lists). Every ordered container becomes an rdf:List bnode spine, which then forces cap special-casing ("list spines are special-cased so the format's own containers don't violate its caps" — gist §5) and inflates the bnode budget the canonicaliser must survive.
2. **Canonicalisation difficulty is self-inflicted.** RDFC-1.0's hard case is bnode labelling; the records are *trees by fiat* (bnode cycles rejected at the gate). Hashing a tree is trivial in any AST formalism. The design first creates the bnode problem by choosing RDF, then contains it with the caps table, the tree rule, and the rev-2 duplicate-FDH rejection — a gate invented specifically because identical sibling subtrees would otherwise "drive HNDQ into factorial permutation exploration inside the caps" (gist §6 step 1). None of these gates would exist over a canonical JSON serialisation (JCS, RFC 8785) of the AST the encoder already consumes.
3. **The duplicate-FDH rule has unmeasured expressiveness costs.** Under a literal reading of RDFC first-degree hashes (all incident quads, adjacent bnodes as placeholders), two structurally identical small filler nodes — e.g. two `[ cdef:ref "1" ]` fillers under `cdef:filler` in different clauses, which the gist's own §3.4 walkthrough record contains several of — share an FDH. Whether the rule as implemented rejects such records I could not verify on this box (the `@jeswr/concept-hash` prototype is not checked out here); the gist's own orchestrator note records that "the implementation had to correct the design's algorithm" in nine deviations (D1–D9). Either reading supports the objection: the RDF path is complex enough that its own reference implementation deviated from its spec nine times, and the safe-subset gate may forbid legitimate repeated substructure [open — verify against DEVIATIONS.md before the arq mint].
4. **Dual-validator drift.** The SHACL shape + RDF grammar checker duplicates what `encoder/src/validate.ts` already enforces on the JSON AST. Two normative checkers for one grammar is a standing divergence surface.

A systems designer without RDF heritage would have said: the canonical form is the typed AST; hash its canonical JSON bytes (NFC + JCS + profile header, exactly the pins §6 already specifies); alpha/ordering canonicalisation is the AST's job (as profile-M's de Bruijn move already demonstrates); RDF, if wanted, is a *projection* for interop — precisely the posture the codegen companion already takes toward OWL/SHACL ("a deterministic projector: snapshot → (OWL/RDFS projection, SHACL projection, labels bundle)", gist §10).

**What the design already answers.** RDFC-1.0 is a W3C REC with a conformance test suite and mandated fail-closed behaviour (gist §2); isomorphism-invariant hashing gives extraction-from-larger-graph invariance (golden vector 11); the sparq estate and the maintainer's federation stack are RDF-native, so the identity layer composes with an existing ecosystem (reports/sparq-estate.md per architecture.md §0.3); and a working prototype with 77 golden vectors exists — sunk engineering that de-risks the path. These are real. But note what they are: *ecosystem* arguments, not *kernel* arguments. Extraction invariance is equally implementable over JSON ("ignore unknown keys at mint"); isomorphism invariance is only needed because RDF introduced bnodes; and the LLM-kernel goal (mandate.md) nowhere requires the identity bytes to be triples.

**Verdict: DESIGN CHANGE RECOMMENDED.** Minimal change: before bead `kernel-of-truth-arq` (mint real ids for kernel-v0) and before any bulk-tier minting at 10⁵ scale, record a coordinator decision on the canonical hash-input form: either (i) a profile whose canonical bytes are NFC+JCS JSON of the typed AST, with the RDF form demoted to a signed projection, or (ii) an explicit, written justification that RDF-form hashing is retained *for estate interop*, acknowledging it as an ecosystem choice rather than a requirement, plus a verified resolution of the duplicate-FDH/filler-node question. Do not let 10⁵ WordNet/Metamath records bake in the heavier pipeline by default. (The hashing *discipline* — content addressing, profile header in-band, multihash agility, caps, fail-closed gates — survives unchanged under either serialisation; nothing in this objection touches it.)

### 1.2 Per-record dereference is linked-data muscle memory; the kernel's consumers want a pack file

**Steelman (the database person).** The serve plane is per-record HTTPS dereference with conneg, immutable caching, SSRF-guarded fetch, redirect refusal (gist §7). But no kernel consumer ever wants one record: the encoder binds *the referenced concept's own canonical vector*, so encoding needs the transitive closure; decode's cleanup lexicon is the whole corpus; E-series training needs the whole vocabulary; the mapper needs the whole label table. Access pattern: bulk, versioned, offline. The natural artifact is a single content-addressed snapshot pack (the lockfile the design already has — gist §8 — made primary), distributed like a tokenizer file or a git packfile. Per-record HTTP dereference optimises for the linked-data browsing pattern ("follow your nose") that no component of this programme performs; it drags in a serving host, guarded-fetch, cache policy, and mirror governance for a use case the kernel does not have.

**What the design already answers.** Snapshots-as-lockfiles exist and are the declared consumption unit for consumers (gist §8); the store is already immutable-direct-200 (no PID/303 layer — the design explicitly rejected the redirect-heavy PID pattern, §7); data/kernel-v0 *is* a pack (concepts/ + manifest with encoder pin) and it is how every experiment consumes the kernel. So the practice is already pack-first; only the *document emphasis* and the deliverable ordering are dereference-first.

**Verdict: DESIGN CHANGE RECOMMENDED (mild, cheap).** Minimal change: specify the **snapshot pack as the primary distribution artifact** (one file: records + manifest + profile bundle hash + prime set, content-addressed) in architecture.md §3 and the prospectus; demote per-record dereference to an optional verification/debug transport. For the ML audience this is not cosmetic: "pip-installable, hash-pinned kernel pack" is a consumable; "dereferenceable URN namespace with federation mirrors" is a semantic-web deliverable they will not integrate.

### 1.3 The ontology axiom vocabulary is OWL heritage with no current consumer — and the bulk tier is about to lean on it

**Steelman.** `cdef:rel ∈ {subClassOf, subPropertyOf, domain, range, disjointWith, metaType, propertyKind, restriction, bridgesTo}` (gist §5) is RDFS/OWL's vocabulary, inherited from the bookmarks/codegen estate. In the actual kernel: **zero** kernel-v0 records carry any axiom (the record shape `{id, label, status, gloss, notes, references, explication}` has no axioms field at all — data/kernel-v0/README.md); molecules-v0 ships `axioms: []` on every record; the encoder scopes axioms out by design (design-molecule-tier.md §4). The one place axioms are about to matter is the bulk tier — 10⁵ WordNet synsets as "AxiomsOnly records (hypernym/meronym/antonym axioms)" (design-bulk-kernel.md). That imports precisely the assumption the programme's own literature review executed: taxonomy-shaped knowledge injected into LMs delivered "narrow, task-local gains, did not move general LM quality, and was effectively abandoned" (nsm-and-knowledge-injection.md §4; Hou & Sachan: "only a marginal amount of knowledge is successfully integrated"). An ontology person reaches for subClassOf because ontologies are made of it; nothing in the E-series consumes it.

**What the design already answers.** AxiomsOnly is explicitly second-class, "no semantic-adequacy claim", visibly flagged inside the hash (`cdef:semanticStatus` — gist §3.1; design-bulk-kernel.md "honesty architecture"); the bulk records' stated purpose is mapper/coverage infrastructure and identity, not semantic claims; and the axiom layer is load-bearing for the *codegen/estate* composition (gist §10), which is a legitimate, separate goal of the maintainer's. The design never claims axioms help the LLM.

**Verdict: RESIDUAL RISK ACCEPTED**, with one guard worth writing down: the prospectus and any lab-facing document must not count AxiomsOnly records toward "kernel concepts the LLM benefits from" — they are identity + mapper substrate. The measured coverage bands already separate "AxiomsOnly-reachable" from "Explicated" (design-bulk-kernel.md); keep that separation in every headline number.

### 1.4 The PL person's challenge: you normalise alpha but refuse every other cheap normal form

**Steelman (the programming-languages person).** Unison — the design's own primary prior art — is not a record format; it is a *language with an evaluator*, and its hashes identify terms whose meaning the runtime can compute. The kernel's "go all the way" version: kernel = the standard library of a small total language; concepts = typed terms; identity = hash of the *canonical normal form*; equality = decidable definitional equality. The design deliberately stops short: "identity is the stated form … even commutativity-normalisation starts an unwinnable 'how much semantics' negotiation" (gist §13 Q1; design-math-sector.md §3.4: `and(a,b)` vs `and(b,a)` mint distinct ids). But the design has already crossed its own line once, correctly: de Bruijn conversion *is* normalisation (alpha-equivalence made syntactic identity), and vacuous-binder rejection is a normalisation-by-prohibition. The "unwinnable negotiation" has a well-known winnable form: admit exactly the class of *terminating, confluent, syntax-directed* rewrites pinned in the profile bundle — e.g. sorting arguments of declared-commutative formers by subterm hash (the Unison ordering-key move applied one level down), double-negation elimination. No entailment, no theorem proving, no negotiation: a decidable, closed list. The cost of refusing it compounds at bulk scale: thousands of mechanically extracted Metamath definitions will mint formulation-variant duplicates that `sameConceptAs` annotations must then repair socially, one signature at a time.

**What the design already answers.** The Lean critique (design-math-sector.md §1.2, §2.4) is decision-grade against *full* definitional equality — identity-modulo-defeq puts a typechecker inside the mint gate, and that argument is correct. The slippery-slope worry is real for *semantic* equivalence. And profile minting is permissionless: a normalising profile-M′ can compete under §8 governance without this document's permission. The disagreement is genuinely open, and the design records it as such (gist §13 Q1).

**Verdict: DESIGN CHANGE RECOMMENDED (scoped, low urgency).** Minimal change: for **profile-M v1** (before Metamath bulk extraction lands), pin a closed rewrite list — canonical argument order for declared-commutative/symmetric formers (`and`, `or`, `iff`, `eq`), double-negation elimination — as a *pre-mint canonicalisation* alongside de Bruijn, with the list content-addressed in the profile bundle. Profile-1 is excluded (NSM clause order is discourse-meaningful; nothing there is declared commutative). This is an hour of spec text now versus a large `sameConceptAs` reconciliation debt later.

### 1.5 The information-theory audit: record overhead is fine; the vector overhead is already owned

**Steelman.** bookmark.json spends ~4 KB of JSON (and more as canonical N-Quads) on ~40 tree nodes over a ~200-atom alphabet — a few hundred bits of content; the caps allow 65,536-byte records for ≤32 clauses; the canonical vector spends 16–64 KB fp16 to carry "~2–4 kbit of structure" (architecture.md §1.2) — an information density of ~1%.

**What the design already answers.** Record verbosity is hash-input ceremony, not a transmitted-per-token cost; SHA-256 doesn't care, the caps keep gates linear, and human-auditable explicitness is the point of the artifact. The vector's low density is the standard VSA robustness trade, correctly derived from published capacity bounds (deterministic-concept-vectors.md §7.2), and the D-vs-d_model mismatch it creates is the panel's unanimous top finding with a pre-registered resolution (architecture.md §1.3). Nothing here is unexamined.

**Verdict: REFUTED.** The lens was run; the overhead is not load-bearing. (Recorded so the refutation is on the books, not skipped.)

### 1.6 Federation/endorsement governance: currently ceremony with zero endorsements

**Steelman.** Every corpus in the repo stamps itself "NOT federation-endorsed"; no endorsement act has ever occurred; yet the governance machinery (endorsement checklists, snapshots, upgrade edges, competing prime sets) occupies a large share of the design surface. For the LLM-kernel goal, governance is a Phase-2+ concern at best; for the research pitch it is noise the two-audience split already quarantines (architecture.md §4, panel O10).

**What the design already answers.** Governance is the *point* of the identity layer for the maintainer's wider estate (ownerless reference — gist §1), it costs the E-series nothing, and the two-pitch structure already keeps it out of the lab-facing story. The "zero endorsements" observation is expectation-appropriate for a day-old programme.

**Verdict: RESIDUAL RISK ACCEPTED.** No change beyond what O10 already decided; the one discipline to hold: governance sections never appear in research-audience documents.

## Lens 2 — the cognitive scientist / lexical semanticist

The catalogued critiques (Riemer's circularity, von Fintel & Matthewson, Bohnemeyer — architecture.md §1.0, nsm-and-knowledge-injection.md §1.3) are NSM-internal. This lens asks the prior question: is *decompositional definition* the right theory of concepts at all?

### 2.1 The Fodorian objection: most lexical concepts have no definitions — and the field agrees

**Steelman.** Fodor, Garrett, Walker & Parkes, "Against Definitions" (Cognition 8(3), 1980) is widely cited as the death knell for decompositional theories *as psychology*: two and a half millennia of definition-hunting produced no uncontested definition of anything ("knowledge", "game", "cause"); the famous test cases fail ("kill" ≠ "cause to die"); and processing experiments show none of the complexity effects definitions predict — even the best candidates for definitional analysis behave atomically ([Fodor et al. 1980](https://philpapers.org/rec/FODAD-3); [IEP on Fodor](https://iep.utm.edu/fodor/); the mainstream reading per [Laurence & Margolis's line of work](https://philarchive.org/archive/LAURCN)). Wittgenstein's GAME (PI §66) is the canonical indefinable: family resemblance, no shared necessary conditions. On this view the kernel's central artifact — the explication — is a category error: a 30-clause paraphrase of a concept that has no decomposition, adopted from the one school (NSM) that never accepted the field's verdict.

**What the design already answers — more than it advertises.** Three genuine answers exist in the corpus: (i) the kernel claims NSM's "engineering value, not its scientific claims" and explicitly not psychological reality (architecture.md §1.0); (ii) adequacy is "social, not proven" — the explication is a *convention to be endorsed*, not a discovered mental structure (gist §1 claim 3); (iii) the scope claim already retreats to "definitional/abstract/relational vocabulary … legal, regulatory, safety, interop" (architecture.md §3, panel O4) — which is, not coincidentally, the register where *stipulated* definition works (statutes define "consent"; nobody discovers it). The honest reframe the documents circle but never state: **the kernel's explications are stipulative, like legal definitions, not descriptive, like psychological theories.** Fodor's argument destroys descriptive definitions; it says nothing against stipulation — a treaty does not need to be a theory of the mind of its signatories. Everything load-bearing (fixity, auditability, structure-derived vectors, shuffled-control experiments) survives under the stipulative reading; only the tacit rhetoric of "capturing" meaning dies.

**Verdict: DESIGN CHANGE RECOMMENDED (documentation, cheap, high leverage).** Minimal change: add one paragraph to architecture.md §1.0 acknowledging the general anti-definitional literature (Fodor et al. 1980; prototype/exemplar work) and stating the stipulative-definition posture explicitly: kernel explications fix *shared reference conventions* whose adequacy is endorsed, not discovered; the programme makes no claim that concepts *have* definitions. This also pre-empts the most likely cognitive-science referee attack on any external pitch.

### 2.2 Prototype/exemplar/graded structure: the kernel is boolean in a graded world — and the molecule tier is already the confession

**Steelman.** Category membership is graded (Rosch; Rosch & Mervis 1975); typicality effects are ubiquitous — including, per Armstrong, Gleitman & Gleitman (1983), even for definitionally crisp categories like ODD NUMBER, which cuts both ways: it rescues a "definitional core + identification procedure" dual theory, but concedes the identification machinery (the part doing the daily cognitive work) is not definitional. Embodied/simulation accounts (Barsalou 1999) place perceptual concepts in modal simulations no amount of prime text reaches. The kernel's own measurements quantify how much of the vocabulary this bites: in TinyStories — "the deliberately favourable ~1.5k-lemma domain" — 33% of top-500 content mass was molecule-gated before molecules-v0, and the open-domain molecule tier "will need to be 10–100× larger" (design-molecule-tier.md §5, M0b). The frequency curve of indefinables does not flatten because the theory improves; it flattens because the tier absorbs it. The molecule tier is decomposition's empirical concession: for perceptual/natural-kind vocabulary, the design already gives up explication and stores a *recognition anchor* (grounding note) — i.e., a prototype pointer — while its rhetoric still frames this as "second-class" ([m]-flagged, "visibly second-class", gist §3.5).

**What the design already answers.** The molecule tier is principled, capped, measured, and honest about concealing complexity (Lemanek cited; design-molecule-tier.md §3); the "mint-only-what-explication-cannot-reach" discipline pushes pressure the right way; and no coverage number merges the tiers. The design *behaves* like a hybrid theory (grounded atoms + relational structure over them) even while speaking pure-NSM.

**Verdict: RESIDUAL RISK ACCEPTED**, with one reframe worth a sentence in the prospectus: the two-tier structure should be presented as the *theory* (grounded lexical atoms + compositional relational layer — the position most consistent with post-classical concept science), not as NSM-plus-regrettable-escape-hatch. Same artifact, stronger story, and it stops the "second-class" framing from implying the molecule-heavy 80% of open-domain vocabulary is a defect.

### 2.3 Polysemy and the sense-enumeration fallacy: one hash per sense inherits a known-broken ontology of senses

**Steelman.** Kilgarriff's "I don't believe in word senses" ([Computers and the Humanities 31, 1997](https://arxiv.org/abs/cmp-lg/9712006)) is the standing result: word senses are task-relative abstractions from citation clusters, not discrete pre-existing objects; every fixed sense inventory fractures on real usage. The kernel mints one hash per word-sense-sized concept; the mapper must then decide, token by token, which discrete record a usage "is" — the ontology problem *and* the WSD problem (conceded as panel O3), now with cryptographic rigidity. The programme's own Amendment A1 is the objection in miniature: five kernel concepts permanently shadowed by collisions, `kind` and `lost` excluded from E1 entirely, ~129 declared wrong-sense tokens injected (poc-design.md Phase M) — in a 1.5k-lemma synthetic corpus chosen *because* it is semantically clean. And the bulk tier is about to ingest WordNet's synset inventory — the exact artifact Kilgarriff's critique was aimed at — as 10⁵ records.

**What the design already answers.** Kernel concepts are *stipulated*, not claimed to partition usage — the mapper maps with abstention, ambiguity is measured not hidden (M0a/M0b), and mapper output is "annotation, never identity" (architecture.md §3). Discreteness at the *identity* layer is unobjectionable (hashes must be discrete); the fallacy would be claiming the inventory exhausts usage, which no document does. WordNet enters as AxiomsOnly with no semantic-adequacy claim (design-bulk-kernel.md).

**Verdict: RESIDUAL RISK ACCEPTED** — the design's abstention-and-measurement posture is the correct response to Kilgarriff, and the risk lives where O3 already placed it (the mapper's coverage/precision at open-domain scale bounds every benefit claim). One guard: WordNet-derived records must never be labelled "concepts the kernel understands" in any external count — they are a legacy sense inventory given stable ids.

### 2.4 The grain question: frames and scripts, not word-sized records

**Steelman.** FrameNet's lesson (Fillmore): meaning attaches to *frames* — buy/sell/pay/cost/price evoke one Commercial_Transaction scene from different perspectives; the word-level records are thin projections of a shared structure. Scripts (Schank & Abelson) make the same point for events. The kernel's unit is the word-sense-sized record; profile-1 has valency frames at the *prime* level (§4.4 — a closed role inventory that is itself frame-semantics machinery) but no shared-scenario object at the *concept* level: give/take, buy/sell, birth/death each re-author their shared scene clause by clause. Costs: authoring duplication, no representation of perspectival relatedness (the encoder's structural similarity will partially capture shared clauses, but only as much as the duplicated text happens to align), and — per X3 — the geometry cannot be trusted to recover it anyway.

**What the design already answers.** Concept references give real substructure sharing where authors use them (18 of 54 kernel-v0 records reference other records; `condolence → grieving → death → event` — data/kernel-v0/README.md); compositionality-by-reference is exactly the frame move at one grain coarser. FrameNet is already on the ontology-mining shopping list (bead `kernel-of-truth-cgh`). And frames are *addable*: a shared-scenario record type referenced by perspectivised concept records is representable inside the existing reference machinery without any new hashing theory.

**Verdict: RESIDUAL RISK ACCEPTED** (design-change candidate deferred): when the FrameNet mining bead executes, evaluate a `ScenarioSchema`-style shared-frame record (one scene record; perspectivised concepts reference it with a role-mapping) rather than flattening frames into per-word AxiomsOnly. File it as a note on `cgh` now; no schema work before then.

### 2.5 What an explication licenses: NSM records are scenario scripts, so "meaning-level verification" overreaches outside the formal sectors

**Steelman.** NSM explications are not necessary-and-sufficient conditions; they are canonical scenario texts ("this someone thinks like this: …"). That partially *shields* them from the classical-definition critique (2.1) — but it also means the A5 claim that verification against definitions "catches meaning-level errors passage retrieval cannot" (architecture.md §4 A5 delta (ii)) is only mechanically true where a checker exists. Profile-P dimension checking is arithmetic; profile-M sort-checking is decidable; profile-1 "verification" against a prototype scenario is a judgement call — exactly the substitutability judgement Riemer showed is circular. The design's sharpest epistemic asset (the formal sectors' "adequacy becomes checkable" — design-math-sector.md §5.1) highlights, by contrast, that the NSM core has no analogous correctness fact.

**What the design already answers.** The gap is stated in both sector documents (math §5.1: profile-1 adequacy "social, not proven"; physics §5: verification is "arithmetic, not another model's opinion" — claimed only for the dimension lane); A5's deltas now carry their own experiment (E9) before any claim hardens (architecture.md §4 rev 2); and the decode–verify loop *does* have a real profile-1 mechanical layer (grammar/valency/referent/caps gates, hash equality, structural comparison) even if adequacy is out of reach.

**Verdict: RESIDUAL RISK ACCEPTED**, with a scoping sentence to add where A5 is described: "meaning-level" verification is mechanical in the formal sectors (dimensions, sorts) and *structural* elsewhere; profile-1 adequacy verification remains social. E9's design should report these strata separately.

## Lens 3 — the ML practitioner who has to consume this

### 3.1 The deflationary alternative: a hash-pinned JSON dictionary of definition TEXT — 80% of the value, 5% of the machinery?

**Steelman, at full strength.** Take the 54+ glosses already sitting in kernel-v0, put them in one JSON file, hash each entry, sign the file. You now have: drift-free reference (hash), O(1) equality, versioning/pinning (the file hash is the snapshot), citations that point at fixed definitions rather than moving text, ZK-friendly commitments (any content-addressed store supports Merkle commitments), and RAG-with-verified-definitions. That is most of the assurance/policy pitch (architecture.md §4, audience 2) — and it needs no prime lexicon, no closed grammar, no SHACL, no RDFC, no VSA encoder, no capacity math, no mapper. The LLM consumes definition text natively — its best input modality — instead of vectors through an adapter that E-series experiments may yet show add nothing. The ripple-effects literature the programme itself cites (Cohen et al. 2024: in-context injection beats ROME by 10–29 points — nsm-and-knowledge-injection.md §6) says *text in context* is the strongest known way to make a model use a definition.

**What the design already answers — and the honest residue.** The specific deltas that survive the deflation, each with its dependency named:
- **Machine-checkable structure** (grammar gates; profile-M sort checking; profile-P's exact dimension arithmetic — 10/10 defining relations EXACT-PASS with a checker that "can actually reject", design-physics-sector.md §4.5). Text cannot be gate-checked. *This delta is real today and is strongest exactly where the kernel left NSM behind* — the formal sectors.
- **Structure-derived vectors with shuffled controls** — the E-series' entire epistemology requires that content and assignment be separable *by construction*; embedded text has no analogous "same marginal statistics, permuted assignment" control at zero cost. But this is a delta for the *research programme*, not for a consumer.
- **Compositional identity** — reference-binding, incremental multiset hashes, cross-record structure. A flat dictionary has none; whether any consumer pays for it is exactly what E4 tier-2 / E8 will show.
- **What genuinely deflates:** the interface/assurance tier. A5-as-product minus the vector layer *is* the hash-pinned dictionary plus grammar gates. The design half-concedes this — A5 is "deployable regardless" and its similarity operation is banned anyway (X3) — but no document prices the deflationary baseline explicitly, and E9 as pre-registered compares decode-verify against "vanilla RAG-with-citations", *skipping the strongest cheap competitor* (RAG over hash-pinned definition text).

**Verdict: DESIGN CHANGE RECOMMENDED.** Minimal change, two lines: (i) add a third arm to E9 — RAG over the hash-pinned definition-text dictionary (same content, no structure) — so the machinery's marginal value over its own deflation is measured, not asserted; (ii) the prospectus names the deflationary baseline as the null architecture and states which deltas are claimed over it (checkability, compositional identity, controls) and which are not (LLM consumption ergonomics). If the kernel cannot beat its own gloss file on E9, the assurance pitch should *be* the gloss file with grammar gates — that outcome must be a legal result of the programme, not an unthinkable.

### 3.2 E2's MET verdict does not yet license the deterministic encoder — the text-embedding baseline is missing

**Steelman.** E2 passed its pre-registered primary criterion in 3/3 model families (partial ρ 0.165/0.214/0.314, all p ≤ 0.0003, random-set 95th-percentile guard passed — poc/e2/results-incoming/20260707-091305-modal/verdict-e2.md). But the covariates were word2vec cosine, WordNet path, and gloss *word-overlap* (poc-design.md E2) — 2013-era relatedness proxies. The obvious 2026 baseline is absent: **embed the explication (or gloss) text with any modern sentence encoder and partial on that RDM.** If SBERT-of-gloss explains the kernel RDM's contribution, then E2 established only that *definitions carry relatedness signal* — known since Hill et al. 2016 and CPAE (nsm-and-knowledge-injection.md §2) — and the deterministic TPR/HRR encoder, the capacity math, and the orthonormal codebooks contributed nothing measurable over `sentence-transformers` on the gloss field. The partial ρ headroom (0.17–0.31) makes this entirely plausible. Compounding: the kernel RDM used is the raw cosine geometry whose pathology X3 documents (afraid↔NOT-afraid at cos 0.9999 vs afraid↔sad at 0.9933; 13/79 inverting edits closer than the nearest genuine pair — poc/results/x3-kernel-v0-report.md), and the pre-registered polarity-stratified reporting is deferred (bead `kernel-of-truth-avt`, runner deviation 11). The kill chain is proceeding on E2's green light (bead `kernel-of-truth-b6z`) with both of these open.

**What the design already answers.** E2 was correctly re-labelled as testing A2's precondition, not A1 (poc-design.md, MAJOR 9); its interpretation is formally "conditioned on X3"; the scope limits forbid strong readings; and adding a baseline RDM is a pure-analysis change — the model activations and item lists are already captured, so this is re-analysis, not re-run. Nothing structural resists the fix; it simply hasn't been named.

**Verdict: DESIGN CHANGE RECOMMENDED (the single highest-leverage item in this review).** Minimal change: before E2 is quoted in any external document or used in any gate decision beyond the already-taken one, re-run the E2 analysis with two added covariates — (a) sentence-embedded gloss RDM, (b) sentence-embedded canonical-explication-text RDM — as a pre-registered amendment; report kernel partial ρ against the *strongest* baseline set; and land the deferred polarity-stratified reporting (`avt`) in the same pass. Cost ≈ CPU-hours. If kernel structure survives *that* partial, the deterministic encoder has its first real external argument; if not, the honest conclusion is that E2-class evidence never required the encoder — text-embedding the explications would have shown the same thing.

### 3.3 Boolean, context-free, frequency-blind: what operations does a consumer actually get?

**Steelman.** LLM semantics is graded, contextual, frequency-weighted; the kernel is none of these. The full operation set a consumer gets today: (1) identity/equality (O(1), cryptographic); (2) decode-given-kernel (100% at depth ≤ 4 at D=8192; 1/54–8/54 at toy-native D — poc-design.md pin note); (3) structural verification (grammar gates; exact dimension/sort checks in formal sectors); (4) membership in a pinned snapshot. **Not** similarity (banned pending an X3-dominating variant — none dominates; bead `kernel-of-truth-5lh`), not typicality, not context modulation, not graded composition, not frequency. That is a *reference and verification* instrument, not a semantic representation — and every use case in the docs that quietly assumed metric structure (kNN in A5, mapper embedding-assist) is already banned by the programme's own rules.

**What the design already answers.** The ban *is* the answer — the design refuses to sell the operation it cannot support (architecture.md §1.2 weakness 1, panel O9), which is the correct posture; relational/structural use is explicitly the surviving mode; A6/E8 (kernel as labelled coordinate system for SAE features) is designed to need labels and identity, not metric trust; and frequency deliberately lives in the mapper/corpus layer, not in identity (a frequency-weighted identity would drift — the exact failure the kernel exists to remove). The boolean/context-free property is not an oversight; it is the price of fixity, paid knowingly (mandate.md defers qualification/accuracy handling by explicit decision).

**Verdict: RESIDUAL RISK ACCEPTED** — with the consumer-facing corollary made explicit wherever the kernel is offered to ML users: the kernel is *complementary* to distributional semantics (identity, audit, verification), never a replacement for it (similarity, gradation, context). One sentence in the prospectus's A5/A6 sections suffices; the internal documents already behave accordingly.

### 3.4 Integration cost vs the consumption modes a lab would actually adopt

**Steelman.** A1 (vocab surgery + mapper + frozen rows) asks a consumer to re-tokenize their corpus through a mapper that is "the ontology problem reborn" (panel O3), pin training to a kernel snapshot they can never hot-swap (update paradox, architecture.md §2 O8), and accept a tied-rows tax — for benefits that E1/E4 have yet to show and 2507.04886 predicts null. Meanwhile the two integration modes with near-zero consumer cost are: (i) **A6/E8** — kernel as a canonical, versioned label space for SAE features (no mapper, no vocab surgery, pure post-hoc analysis); (ii) **A5-as-eval** — semantic regression testing between model versions against pinned definitions (an offline harness). The design already knows this (A6 was *added* as the fallback room-mover; the panel's "one result that moves the room" is E8) — but the build order still runs the expensive-integration experiments first and the cheap-integration ones as fallback.

**What the design already answers.** The ordering is deliberate and cheap: the kill chain prices A1's question at $30–80 *because* it is the maintainer's original hypothesis and dies cheaply if false (architecture.md §0.4); E8 is staged as the pivot, not the afterthought; and "do not approach a frontier lab before E7-or-E8" already encodes the consumption reality. The objection reduces to emphasis, and rev 2 largely fixed it.

**Verdict: REFUTED** (as a design objection — the staged-bets structure is the right answer; the consumer-cost analysis it implies is already distributed across O5/O6/O10 and the two-pitch split).

### 3.5 Determinism as a virtue: audited, cheap, and correctly de-fanged

**Steelman.** Determinism is the encoder's headline property, but the security-relevant boundary was conceded to be the *signature*, not re-derivation ("verification-by-re-embedding is advisory diagnostics only … the security boundary is the signature, full stop" — gist §9a); and every trained-model touchpoint (A2 adapters, E8 alignment) passes through learned, seed-dependent components anyway, so end-to-end determinism never reaches the consumer.

**What the design already answers.** The overclaim was already withdrawn (gist §9a rev 2); what determinism still buys is real and cheap: exact reproducibility of every Phase-X/E-series artifact (encoder content-hash pinning, X0 goldens), no fit-corpus provenance questions, and re-derivability that lets the 10⁵-record bulk tier store zero vectors (design-bulk-kernel.md: vectors on demand, "the encoder is deterministic so vectors are always re-derivable"). Costs approximately nothing (SHA-256-seeded codebooks), bought honestly.

**Verdict: REFUTED.**

## The honest answer to the maintainer: "how much of this is my semantic-web bias?"

Less than you fear at the core, more than you'd like at the edges — and almost all of it is priced, not hidden. The load-bearing ideas (content-addressed identity, fixity of reference, closed capped grammars, deterministic encoding, fail-closed gates, staged falsification) owe more to Unison, Nix, and property-based engineering than to RDF, and they survive de-RDF-ing untouched. Your own honesty architecture is the countervailing force (the scope claim, the molecule concessions, the X3 ban, the kill chain), and it is winning on most fronts.
