### H-CFT — closure-distillation fine-tune (bake-vs-retrieve on materialised entailments)

**One line:** Materialise the deterministic closure of the default world (sparq-reason, proof traces attached) into training data and LoRA-tune a tiny host, with a paired retrieve-the-closure arm, to test whether engine-derived facts internalise better than they retrieve.

**Thesis:** EFFICIENCY primary (store→weights compression; zero per-query engine cost on covered entailments), CORRECTNESS secondary (does entailment competence transfer at all).

**Sketch:** Use the RULES-1 estate: kinship TBox (data/axioms-v0 + axioms-kinship-v1), world-v0 + nsk1-clutrr ABoxes, sparq-reason materialisation with why() proof trees. Build four matched fine-tune corpora for SmolLM2-135M: (a) stated facts only; (b) stated + materialised entailed facts; (c) stated + DERANGED/shuffled-rule closure (the alignment/content control, the knull lesson); (d) stated + proof-trace CoT renderings. Grade zero-shot on held-out ENTAILED queries (held-out relation compositions and proof depths, third-party CLUTRR gold — the D4 diagnostic splits). Paired H-RB arm: same closure served by retrieval at inference at matched KOT-SIZE/2 bytes. Decisive kill: (b) ≤ (a) or (b) ≤ (c) on entailed cells kills baking; (b) ≫ retrieval arm at matched bytes is a W2-relevant DIAGNOSTIC. Formal/templated inputs only → oracle-diagnostic label per ASM-0814, no NLB gate needed, no W1 claim.

**Upstream deps:**
- sparq-reason closure + why() proof trees (local checkout /home/ec2-user/css/kernel/sparq; SPARQ–twin agreement 1207/1207 already certified)
- RULES-1 materialisation pass + CLUTRR third-party gold (poc/rules-1, data/nsk1-clutrr)
- RULES-1 GPU host-lift grade (informs priors, does not block)
- P3-D-BASE tuning-symmetry rules if promoted beyond diagnostic
- P3-LR-NTP proof-trace-distillation lit review (co-input, not a blocker for the S-scale run)

**Rough cost:** S–M: corpus generation ~$0 CPU; 4×LoRA on 135M ≈ 5–20 GPU-h, fits the free pool (Lightning/Modal).

**Novelty:** The portfolio names proof-trace distillation only as an H-GU training variant and bake-vs-retrieve only as the H-RB rider; no concrete experiment exists anywhere that trains on engine-materialised closure. This is the 'world-model closure materialiser feeding a fine-tune' slot, instantiated on already-certified materials with a deranged-closure control the existing designs lack.

### H-PCA — proof-carrying acceptance store (verify-retry v2)

**One line:** Replace f2b's string-equality answer-key acceptance with engine re-derivation: a claim is accepted iff sparq-reason re-derives it from stated facts and the checked proof object, extending the measured +0.25 verify-retry mechanism to entailed items where no flat key can exist.

**Thesis:** EFFICIENCY primary (the one measured-positive mechanism, extended past its coverage ceiling), CORRECTNESS secondary (dissolves the answer-key confound by construction).

**Sketch:** Productise the f2b loop with a new acceptance predicate: host answers; the engine attempts to derive the answer from the stated ABox+TBox; accept iff derivation succeeds (proof trace logged as provenance), bounded retry/abstain otherwise. Run on the RULES-1 entailed-cell items — exactly the cells where DECONF-A1's C_dec=0.0 proves a stated-fact key cannot reproduce the decision, so any lift is NOT attributable to an aligned answer key (KOT-FAIR/2 confound B answered structurally, not by controls alone). Arms: 135M+proof-accept vs 135M alone vs 135M+stated-key f2b loop vs shuffled-rules acceptance (must collapse) vs 1.7B alone. Decisive: LCB95 lift on entailed cells > 0 with shuffled-rules collapse = the first kernel-machinery-attributable host lift in the registry; null = the verify-retry mechanism is key-bound, a major steer. Formal inputs → oracle-diagnostic; the NL leg waits on NLB.

**Upstream deps:**
- f2b runner (poc/f2b/runner/f2b_runner.py) + its registered controls
- sparq-reason why() + the RULES-1 CPU certificate harness
- RULES-1 GPU A3-vs-A1 grade (its injection arm is the natural sibling; sequence after it lands to avoid duplication)
- knull rules-source ablation for kernel-specific attribution (pending, phase-2)

**Rough cost:** S: acceptance predicate is CPU-µs; host runs ≈ 2–8 GPU-h on 135M; ~$0–50.

**Novelty:** F2b accepts by string equality against an aligned key; the RULES-1 GPU arm INJECTS derived facts into context. Nothing in the portfolio uses proof-checking as the acceptance predicate. It is also the cheapest structural answer to the review's deepest confound (§2.2 answer-key problem) — the win class it enables is one the current F2b design can never license.

### H-KVP — training-free kernel-vector soft prior (deterministic encoder as rerank/logit head)

**One line:** Use the validated deterministic explication→vector encoder (the programme's canonical-concept-vector asset, currently unused in any host integration) as a training-free decode-time reranker/logit prior over candidate answers and records.

**Thesis:** CORRECTNESS primary — this is the purest test of the founding thesis that training-free grounded concept vectors can do runtime work for a host.

**Sketch:** For each covered query, encode the query's kernel explication and each candidate answer/record with encoder/ (Hadamard-TPR + HRR construction, content-hash pinned, X0–X4 validated); rerank the 135M host's top-k samples (or bias logits over a candidate set) by vector similarity. Zero training — sidesteps nsk1's measured wall (text grounding net-harmful; trained residual bridge stuck at echo grade). Mandatory comparators at matched bytes/cost: BM25 rerank, a generic small sentence-embedding rerank (the matched generic-RAG control class), self-consistency at equal k; controls: deranged concept→vector assignment, shuffled lexicon primes. G1 first: coverage-ceiling Δ_max on paper, ~$0 — expected to bind hard, and that is informative. Decisive kill: kernel-vector rerank ≤ generic embedding rerank on the covered slice = deterministic grounded vectors add nothing over learned embeddings at this seam, a direct adverse datum for the founding CORRECTNESS story at S cost.

**Upstream deps:**
- encoder/ package + Phase-X goldens (X0 byte-determinism, X4 JL projection)
- kernel-v0/molecules-v0 explication coverage census (G1 input; coverage 0.3542 wall applies in full)
- P3-D-RAGC matched generic-embedding control spec (can be drafted locally)
- no sparq or training dependency

**Rough cost:** S: CPU encoding + rerank; host sampling ≈ 1–5 GPU-h.

**Novelty:** H-GNN-LF is the nearest portfolio neighbour but presumes a TRAINED graph encoder; nsk1 used trained residual writes. No portfolio member ever connects the encoder/ artifact to a host. This fills the 'quoted-triple/explication-vector kernel embedding as a soft prior' slot in its cheapest, training-free form.

### H-ITN — in-tokenisation/normalisation layer (the a-e2 consumption channel)

**One line:** A deterministic input-edge canonicaliser that rewrites covered surface phrases to pinned canonical forms (variant B: reserved concept tokens with embedding-only tuning), finally measuring the CONSUMPTION side of the a-e2 prefill-membership upper bound.

**Thesis:** EFFICIENCY primary (prefill token/cost compression at matched accuracy), CORRECTNESS secondary (normalisation as a cheap rules-in-LLM placement upstream of every other family).

**Sketch:** Variant A (training-free): longest-match recogniser over the minted concept membership lists rewrites covered spans to a single canonical paraphrase; stock host consumes the normalised prompt. Variant B: reserved single tokens per concept, embedding-only tuning (base weights frozen). Measure on covered slices of the R-1 suite: prefill tokens, KOT-COST/2-style cost, accuracy non-inferiority (single one-sided CI vs a pinned margin, per §2.5(4) — not TOST). Controls: random-synonym rewriting at matched compression (does any shortening help/hurt equally?), deranged concept→token assignment for variant B. Decisive: realised savings as a fraction of the a-e2 membership bound (18.5–41.7% upper bound) at non-inferior accuracy; accuracy break or ≪10% realised savings kills the channel and retires the a-e2 bound from the efficiency ledger. CASC-0′'s gloss≈plain result is a registered adjacent caution (outbound dialect), not this seam — this is inbound compression, which CASC never measured.

### H-GWP — grammar+TBox-guarded world population (constrained-decoding NL front-end)

**One line:** Attack the programme's measured NL wall by emitting world-records only through engine-constrained decoding over the closed record grammar with load-time TBox type-checking as a hard reject — a concrete candidate for clearing the NLB gate that everything else is blocked behind.

**Thesis:** CORRECTNESS primary (the checking engine guarding the parse; kills the S2 dangerous-wrong mode by construction), EFFICIENCY downstream (every store-touching family's NL leg is blocked on exactly this gate).

**Sketch:** H-PS × H-RULE-CD hybrid at the input-population seam: the host parses NL into ABox records under grammar-masked decoding (only well-formed records are emittable); each record then passes kot_axiom/sparq-reason load-time validation (domain/range, functional, disjointWith) — ill-typed facts are rejected at emit time, with calibrated abstention when no valid parse clears a confidence bar. Rerun the l3a-parse and a5-nl materials verbatim for direct comparability against the measured baselines (retention 47.6%/41.6%; a5-nl S2 dangerous-wrong 5.0%). Decisive: S2 rate → ~0 by construction is checkable cheaply; the open question the run answers is whether retention reaches the pinned 0.90 NLB bar or whether type-rejection buys safety only by collapsing coverage (the risk-coverage curve is the deliverable either way). A PASS unblocks the NL legs of H-VL, H-PCA, H-RULE, H-GNN simultaneously — highest option value per dollar in the portfolio.

**Upstream deps:**

**Novelty:** H-PS exists as a family and NLB as a gate, but no concrete design or experiment exists; the portfolio's constrained decoding (H-RULE-CD) sits at the ANSWER edge. Using the engine's validator as an emit-time reject filter on the PARSE edge is unoccupied space and is the only candidate that directly widens the whole programme's bottleneck.

### H-DCB — entailment delivery-channel bake-off (text-prepend vs KV-inject vs proof-accept)

**One line:** Hold the derived facts fixed and vary only the delivery channel into the host — verbalised text prepend, KBLaM-style KV injection, and verify-retry acceptance — to decide once, cheaply, where engine output should enter a model.

**Thesis:** Both, as a decision instrument: it does not test a new mechanism, it ARBITRATES between the portfolio's competing integration seams with one paired design, steering all subsequent EFFICIENCY and CORRECTNESS spend.

**Sketch:** Take the identical per-item entailed-fact sets from the RULES-1 materialisation (temporary-world delta closure for prompt-supplied facts, per WMRE-1's two-regime design). Three delivery arms at matched token/resource budgets on the same CLUTRR entailed items, same 135M host: (1) text prepend (the nsk1-g2d channel, measured net-harmful for glosses — does it also harm answer-bearing derived FACTS, or was the harm gloss-specific?); (2) KV injection keyed by content-hash (instantiates H-RULE-KV, with causal-ablation provenance per ASM-0815 — attention maps disqualified); (3) verify-retry acceptance (the measured-positive seam). Shared controls: shuffled-fact injection, irrelevant-but-structurally-matched records. Paired per-item predictions → hierarchical bootstrap, one primary endpoint (channel × accuracy interaction). Decisive by design: whichever channel dominates the Pareto (accuracy, p95 latency, resource vector) sets the default integration seam for H-CFT/H-PCA/H-RULE and retires the others to documented-second-choice status; a text-prepend win would also cheaply overturn the nsk1 caution's generalisation.

**Upstream deps:**
- RULES-1 materialised entailed-fact sets + GPU harness (poc/nsk1/nsk1_runner.py lineage)
- sparq-reason incremental delta closure for the temporary-world regime
- a minimal KV-injection adapter (KBLaM-style; the only trained component, small) — P3-LR-RULE lit review co-input
- nsk1-g2d/bprime2 assessments as registered adjacent evidence
- oracle-diagnostic labelling (ASM-0814): gold-parsed worlds, no W1 claim

**Rough cost:** M: arms (1) and (3) ≈ 5 GPU-h inference; the KV adapter ≈ 10–30 GPU-h training+eval; total ~15–40 GPU-h in the free pool.

**Novelty:** The portfolio lists these seams as separate families (H-RULE-KV placement table, H-VL, WMRE injection arm) but never pits them against each other on identical content — the head-to-heads currently planned are family-internal. One paired experiment replaces three sequential family runs and converts the nsk1 caution from a standing fear into a measured, scoped datum.

## Design (agent `design:H-PCA`)

### Architecture

Five components wrapped around a frozen small host, replacing f2b's string-equality-vs-aligned-key acceptance with engine re-derivation. (1) HOST: SmolLM2-135M (frozen weights, no training) answers from stated facts only. (2) WORLD COMPILER: per-item ABox world (gold-parse, ASM-1123) + pinned content-addressed TBox (kot-axiom/1 records, each citing its endorsing NSM explication sha; axioms-v0 + axioms-kinship-v1) + item-scoped UNA sidecar (ASM-1120). (3) ACCEPTANCE ENGINE: accept(claim) iff the claim is re-derived from Cl(stated ABox + TBox) with a proof tree whose every node carries a regime tag from the EXPR-1 ladder (owl-rl / policy only in v1 — L-A plus the single L-C R-COVER-ELIM rule; NO new outer-layer rules, per the expressivity-analysis.md ladder discipline: every rule sits at the innermost sufficient layer, and the kinship inventory audit PROPOSED-ASM-1236 finds nothing above L-C needed) and whose leaves are exclusively stated facts, endorsed axiom records, or named registered ASM premises (UNA/covering visibly in every why() tree). Off-coverage or underivable → licensed rejection with named ERR_* code, never approximation (fail-closed, ERR_INSUFFICIENT_PREMISES / ERR_RULE_UNIMPLEMENTED). Engine = sparq-reason reason_n3 (primary, commit 0ab87b2) + the ~200-LOC Python differential twin (twin_engine.py) which MEASURED-agreed with sparq-reason on Cl(S) for all 1,207 conformance cases; the twin doubles as an independent proof-object CHECKER (structural validation of each step against the closed rule inventory), which is the v2 increment distinguishing "derivable" from "this specific proof object checks". Acceptance is CPU-microsecond scale (certificate: 0.4 s for 958 items + 248 cells on 2 shared cores). ### Grounding mechanism

# H-KVP — training-free kernel-vector soft prior (deterministic explication→vector encoder as a decode-time reranker over host candidates; logit-prior variant deferred)

## Design (agent `design:H-KVP`)

### Architecture

Five components, zero training anywhere. (1) HOST: pinned SmolLM2-135M-Instruct (the f2b/DECONF checkpoint), unmodified; it produces candidates only. (2) CANDIDATE GENERATOR, two variants: KVP-C (first form) scores a CLOSED candidate inventory — the covered answer set for the task family (e.g. the CLUTRR relation vocabulary / f2b answer slots), where each candidate IS a kernel-v0/molecules-v0 concept, so candidate→explication is exact and lossless; KVP-R (second form) reranks the host's k temperature samples (k=8–16), mapping each sampled answer to a covered concept via mapper/ (abstain-and-record discipline; unmappable candidates fall back to host score). (3) KERNEL SIDE: per covered eval item, an item-aligned query explication AST (kot-ast/1; built via mapper/ where the surface is covered, authored otherwise — self-authorship rider disclosed, exactly f2b's aligned-key regime); encoder/ encodeExplication → 8192-D unit Float64 vector, encoderContentHash-pinned; each candidate's canonical concept vector via memoised encodeConceptSet (cyclic refs fail closed). (4) SCORER: cosine similarity in 8192-D; decision rule argmax_c sim(v_query, v_c); sub-margin ties fall back to the host's own choice (disclosed fail-open at the seam) or abstain under the honesty-first scoring regime. This is a SCORING POLICY, not an inference rule — per the expressivity-analysis ladder (docs/next/arch/expressivity-analysis.md §4.2) H-KVP sits below L-A, invokes no owl-rl/horn-def/policy machinery, needs no new regime tag, and its outputs must never be reported as entailment. (5) COVERAGE ROUTER: an item enters the kernel arm only if mapper coverage + explication availability hold (the 0.3542 molecules-v0 wall applies in full); uncovered items route to host-alone. DEFERRED OUTER LAYERS (gated on the CDE): a logit-prior variant at host d_model=576 using the X4-validated JL projection or the pre-registered kot-enc-Bq/1 native-D encoder; and record-rerank over RDF worlds via sparq .spqv. Topology note: this is deliberately a SELECT topology — the one delivery family measured net-positive at tiny hosts (f2b +0.2507 MEASURED VERDICT-GRADE) — and avoids the text-delivery channel measured net-harmful (nsk1 −22/−31pp, MEASURED-exploratory) and the trained residual bridge stuck at echo grade. The host never reads kernel content; the kernel judges candidates.

### Grounding mechanism

Deterministic compositional geometry, not learned association. A concept/query vector is a pure function of its NSM explication's structure over the 65-prime lexicon: exact Hadamard-TPR binding within clauses (orthonormal Sylvester–Hadamard codebook, zero crosstalk), whitened unitary circular-convolution HRR across clauses/depth, all structure derived from SHA-256 over fixed labels — no seeds, no corpus statistics, no gradients; byte-identical across runs/platforms (X0 goldens MEASURED). 'Grounded' is operationalised falsifiably: similarity between vectors reflects shared explication structure (X1 single-edit margins MEASURED — on SYNTHETIC explications only, a disclosed gap), and grounding is load-bearing at the seam iff the rerank win collapses under (C1) deranged concept→vector assignment (permute which concept gets which vector: preserves geometry, destroys grounding) and (C2) shuffled-lexicon-prime encoding (destroys the prime semantics, preserves the construction). If A4 wins but C1 matches it, the mechanism is generic vector-space separation, not grounding — and must be reported as such.

### Thesis targeted

CORRECTNESS, primary — this is the purest available test of the founding thesis that training-free, NSM-grounded canonical concept vectors can do runtime work for a host: no training, no bridge, no prompt-channel confound; the deterministic vector either selects better answers than matched learned/lexical alternatives on the covered slice or it does not. Mechanism of help: convert host sampling/candidate headroom into accuracy by choosing the candidate whose grounded meaning-vector best matches the query's explicated meaning — the kernel occupies the engine-owns-selection seat that f2b/DECONF measured as the surviving tiny-host value location, while directly attacking DECONF-B's open wound: no kernel-SPECIFIC runtime residue has ever been measured (GS-A–kernel identity 1.0). EFFICIENCY: descriptive only (the prior is ~$0 CPU at inference); no efficiency claim is licensed by any outcome here, and none is registered.

### Upstream sparq work

Named optional bands, all gated on CDE survival: [S, 1–3 days] a KernelEmbedder adapter implementing sparq-vectors' out-of-process Embedder trait (embed.rs) so deterministic kernel vectors land in .spqv rows keyed by dict-term id — the store is id-keyed so this needs no schema change; buys ANN (exact/DiskANN/HNSW) over kernel vectors for the record-rerank variant, with the v3 provenance header carrying encoderContentHash as embedding provenance. Explicitly NOT needed: the KGE trainer, the gufo prior, the taxonomy block — H-KVP is training-free by definition and must stay so.

Control interpretive gap (disclosed but worth sharpening): A4>C1 and A4>C2 do not establish grounding-specificity beyond A3 -- C1 destroys ANY consistent aligned readout, generic or grounded; the design's own if-C1-matches-report-as-generic clause is the right discipline, but then the grounding claim rests entirely on A4-A3, doubling the weight on the underspecified query-explication channel.

# H-ITN — in-tokenisation/normalisation layer (the a-e2 consumption channel)

## Design (agent `design:H-ITN`)

### Architecture

A deterministic canonicaliser interposed between raw prompt text and the host tokenizer; the host model is otherwise stock. PIPELINE (both variants share stages 1-3): (1) RECOGNISER — a longest-match trie over the pinned per-language surface lists from the a-e2 census (poc/a-e2/results/per-concept-detail.jsonl + en-mhat.jsonl; a1-hybrid mapper policy sha e13dc838..., kernel-v0 manifest da56bf1f...), scanning the prompt left-to-right, matching only whole word-boundary spans, restricted to wordlike=true surfaces (ASM-0562 universe) whose census decision-kind is prime|concept (never abstain) — fail-closed: any uncovered or ambiguous span passes through verbatim. (2) REWRITE TABLE — a pinned surface→canonical map frozen by content-hash before any measurement. (3) PROVENANCE EMITTER — every rewrite appends a span record (byte offsets, original surface, concept/list id, canonical form, table hash), making the transformation auditable and exactly invertible from the record; this is the same fail-closed + provenance discipline as the RULES-1 engine, applied at the input edge. VARIANT A (training-free): the canonical form is a TEXT surface — for English spans, the cheapest-host-token synonym within the same census cluster (small gains, en bound only 3.9-5.9% @10k wordlike); for the fertility-bearing non-English spans (fi/ja/es, where the a-e2 value actually lives), the English canonical surface obtained through a pinned public bilingual dictionary (MUSE-class), declared as an A-F1-PROXY instrument [STIPULATED at freeze; alignment quality is itself unmeasured and bounded by ASM-0462's bracket rules]. The stock host consumes the normalised, partially code-switched prompt. VARIANT B (embedding-only): one reserved single token per covered concept appended to the host tokenizer vocabulary (top-N by value_membership, N≈1k-3k), embedding row initialised as the mean of the canonical surface's constituent-token embeddings, then embedding-rows-ONLY tuning on a small covered corpus with every base weight frozen; per-language surfaces map to their token directly, so variant B realises the non-English bound WITHOUT the A-F1 alignment fork. LADDER JUSTIFICATION (per docs/next/arch/expressivity-analysis.md discipline, applied to machinery rather than entailment since H-ITN performs no inference): the cheapest layer that suffices is a deterministic pinned-table longest-match rewrite — no learned NER/linker, no context model, no entailment regime is adopted; any escalation (context-sensitive disambiguation) is gated on the measured abstain/error profile of this deterministic layer, exactly parallel to the L-A-first rule. NOVELTY vs the portfolio: all H-RULE placements (CD/KV/AD/ACT/HL, programme-3 §3.3) sit at decode/KV/activation seams; nothing sits before the tokenizer, and the a-e2 bound is explicitly quarantined 'consumption channel unmeasured' — H-ITN is that measurement.

### Grounding mechanism

Grounding is by pinned symbolic identity, not learned association. (i) Every rewrite is licensed by membership in a content-hash-pinned surface list descending from the a-e2 census artifacts (sha-pinned wordfreq 3.1.1 lists, pinned SmolLM2-135M-Instruct + Qwen2.5-0.5B tokenizers, pinned mapper policy), with the census abstain set (164 en abstains) hard-excluded. (ii) The canonical form per concept is frozen before measurement and carried in the rewrite table hash; variant B keys each reserved token to its concept id so the token IS the concept's fixed address. (iii) Span-level provenance records make every consumed prompt reconstructible and every accuracy failure attributable to a named rewrite — the normaliser is inside the measured system per the ASM-0808 NL-input-integrity rule, so its mis-rewrites are the system's failures by design. (iv) The grounding is TESTED, not assumed, by two controls: random-synonym rewriting at matched compression (does concept-pinned canonicalisation differ from arbitrary shortening?) and, for variant B, a deranged concept→token assignment (if a scrambled map performs identically, the benefit is generic tokenizer extension — the K-A3 null — and kernel grounding at this seam is refuted). HONEST QUARANTINE [STIPULATED]: no 10k-concept kernel mint exists yet (A-E1 is gated on A-F0 + a K-A4 pricing instrument), so the census lists act as PROXY concepts; ITN-0 therefore measures the CHANNEL (can canonicalised/reserved-token input be consumed at non-inferior accuracy, and what fraction of the membership bound is realisable), while kernel-ATTRIBUTION claims remain gated on the mint and the deranged control. The registered adjacent caution is carried: CASC-0' found gloss≈plain on the OUTBOUND dialect (registry/verdicts/casc-0.json, INSTRUMENT-INVALID overall, K3' attribution kill at its scope) — that seam is different from inbound compression, which no CASC cell measured, but it warns against expecting kernel-specific magic beyond the mechanical token arithmetic.

### Thesis targeted

EFFICIENCY primary: mechanism of help is prefill-token/cost compression at matched accuracy — the a-e2 census established [MEASURED-exploratory, membership UPPER bound, registry/assessments/a-e2-census.json] a blended selection-stage pool of 18.5-24.0% (Qwen2.5) / 33.4-41.7% (SmolLM2) @10k concepts (uniform, wordlike), concentrated in non-English fertility cells (fi 38.7-44.3%, ja up to 56.7%); H-ITN converts that quarantined bound into a REALISED-savings measurement, the one number the efficiency ledger is missing. # H-GWP — grammar+TBox-guarded world population (constrained-decoding NL front-end)

## Design (agent `design:H-GWP`)

### Architecture

Three-stage fail-closed NL front-end replacing the measured-FAIL deterministic a1-hybrid pipeline at the input seam, engine untouched. STAGE 1 — grammar-masked emission: a small host LLM (SmolLM2-135M/1.7B; Qwen2.5-7B diagnostic ceiling) decodes the NL input directly into a kot-query/1 (or kot-world/1 ABox) record under engine-derived token masking: the closed record grammar (tools/axiom/kot_axiom.py validate_* functions define it exactly) is compiled to an xgrammar/outlines-class CFG whose terminal vocabulary for URN slots is the PINNED lexicon + entity gazetteer rendered as a trie — only well-formed records over licensed URNs are emittable, plus one mandatory ABSTAIN production (a refusal terminal) so constrained decoding cannot force an answer on out-of-scope inputs (without this, S1 control-refusal collapses by construction — a load-bearing design point). This is grammar/closed knowledge in the ASM-0146 sense, never answer knowledge; per ASM-0815 the masking is transport — the grammar+lexicon derivation, not the LLM, is what guarantees well-formedness. STAGE 2 — calibrated abstention: temp-0 greedy decode; the sequence log-prob margin over the constrained support is thresholded at tau, set once on the frozen 60-item/vertical DEV sets (never on EVAL); below tau emits ERR_PARSE/low-confidence. The risk-coverage curve over tau is a primary deliverable regardless of verdict. STAGE 3 — engine validation as hard reject: the emitted record passes kot_axiom load-time checking exactly as today (ERR_BAD_QUERY, ERR_TERM_UNLICENSED, ERR_UNKNOWN_ENTITY on the query seam; VIOLATION_FUNCTIONAL/DISJOINT/DOMAIN/RANGE surfacing with ERR_CONFLICT on the world-population seam) — an ill-typed fact never enters the store; a rejected record is a refusal with a stage tag, never a guess. Ladder discipline (expressivity-analysis.md section 4.2): H-GWP adds NO entailment layer — the engine stays L-A owl-rl + L-C policy; the front-end sits before the entailment boundary entirely, so no gap-code justification is owed; it is the FK-NLB-3 pre-declared LLM-assisted-parse successor, which the frozen records re-opened when the kill fired. Registry discipline: new record ids (l3a-parse-cd, a5-nl-cd) since the parent front-end pin is superseded; frozen corpora, stores, engine pins, and analysis-boundary constants inherited verbatim. What is deliberately NOT claimed: constrained decoding cannot repair the FK-NLB-10 shape-ambiguous strata (unique-maker/made-lookup: the distinction is absent from unmarked English — an information-theoretic limit, not a parser limit), so the l3a primary keeps the n=527 7-family scope; and a well-typed but mis-bound record survives masking, so S2 is narrowed by construction, not zeroed — the residual is gated, not assumed away.

### Grounding mechanism

Grounding is by construction at three nested levels, each with exact provenance: (1) SYNTAX — the decoder's reachable set at every step is derived from the closed record grammar, so no out-of-grammar byte sequence is emittable (the grammar derivation is the provenance for well-formedness); (2) VOCABULARY — URN slots are trie-constrained to the pinned kernel/molecule/code manifests plus the world-store entity gazetteer, so every emitted term is a licensed kernel concept or an asserted entity, never a hallucinated identifier; (3) SEMANTICS — the engine's load-time TBox validation (domain/range via disjointness, functional, disjointWith, cardinality, CWA violation surfacing) rejects type-inconsistent records before any store write or query execution, and every answer that does flow carries the engine's provenance set + license string exactly as in the frozen parents. The LLM contributes only the mapping from open English to a point in the pre-validated record space, with calibrated abstention when its confidence over that space is low; everything downstream of the emit boundary is the existing deterministic, microsecond, proof-carrying engine. ### Failure modes

1. Registry collision / instrument fork: the FK-NLB-3 successor slot is already occupied by the registered NLB/1 design (ASM-0900, docs/next/design/NLB.md, dated 2026-07-11, GPT-5.6-reviewed to rev-2); minting parallel record ids l3a-parse-cd/a5-nl-cd forks the gate instrument P3-E-NLB-1 and re-litigates decisions (corpus protocol, calibration splits, acceptance arms) the registered design already settled after review.
2. Non-decisive 'decisive' experiment: rerunning the byte-identical parent K=1 corpora is pre-declared diagnostic-only — ASM-0904(1) ('results can never substitute for the gate'), ASM-0944(1) (legacy corpora are public, design-contaminated, K=1, explicitly NON-DECISIVE), ASM-1092. A PASS on it clears nothing: G-NLB (ASM-0901) is only clearable on NLB-EVAL/1 (ASM-0902), so the claimed simultaneous unblocking of H-VL/H-PCA/H-RULE/H-GNN/H-DD natural-input legs does not follow from the proposed run.
3. Calibration under-powered by ~an order of magnitude: tau set on 60-item/vertical DEV sets cannot place a threshold against the S2 bound (dangerous-wrong Wilson UB < 0.02 on n=527 permits only ~4-5 wrong answers; a 60-item set cannot estimate a 1-2% risk region). NLB/1 rev-2 replaced exactly this with disjoint CAL-A/CAL-B ~20%+20% query splits (ASM-0942(1)) after review flagged the role violation.
4. Constrained-support confidence inflation: renormalizing log-probs over the masked support makes the model look confident whenever the licensed continuation set is small, regardless of semantic grounding — the sequence log-prob margin is a biased selector for exactly the S2 mis-binding class the gate cares about; the proposal's sole reliance on tau + TBox rejection drops NLB/1's pre-registered contract-inversion and ambiguity-set-execution acceptance arms (ASM-0905(5)), which exist because confidence thresholds are known not to catch consistent confident role/direction inversions (both orientations of mother-of/father-of are well-typed, so Stage-3 TBox validation passes them).
5. Local-vs-global mask infidelity unaddressed: the proposal compiles to 'xgrammar/outlines-class CFG' without the mandatory exact product-automaton (grammar x tokenizer x trie x type-constraints) local-vs-global equivalence check (ASM-0905(3)/ASM-0947(2)); local token masking can admit prefixes with no valid completion (dead-ends force either malformed truncation or spurious ABSTAIN) and distorts the very margins Stage 2 thresholds.
6. ABSTAIN-production pathology at temp-0: a single refusal terminal competes against the summed mass of the entire answer subtree; greedy argmax over that geometry tends to either never or always abstain depending on prompt phrasing, and the margin definition over 'constrained support' including ABSTAIN is unspecified — the load-bearing S1 design point is stated but not operationalized.
7. a5 vertical is currently design-blocked (ASM-1342, measured 2026-07-12): the container-ask two-op ambiguity is a corpus/protocol defect the parents measured; constrained decoding cannot disambiguate a genuinely ambiguous NL surface, so the a5-nl-cd leg inherits the block and its S2 budget fails at source until the annotation-channel/phrasing-protocol revision lands.
8. Resource-accounting breach: SmolLM2-1.7B fp16 (~3.4GB) exceeds the ASM-0946 packed-artifact ceiling of 1.0GB unless quantized (quantization then needs its own determinism/G6 byte-identity story on GPU — vLLM batch nondeterminism must be pinned batch-1); Qwen2.5-7B is safely framed as never-gated diagnostic.
9. Capability-limited hypothesis may fail low: PARSE lit review rates capability-limited as leading-but-unresolved for the frame-miss mass; a prompted (not fine-tuned) 135M-1.7B model may sit below even the deterministic mapper's covered exactness, burning the at-most-one-per-vertical NLB-0 diagnostic slot (ASM-0904(1)) that the registered fine-tuned Tier-1 pilot was budgeted for.

### Rationale

H-GWP is well-aimed at the programme's one measured wall (l3a-parse LB 0.4407, a5-nl LB 0.3889, both FAIL) and its three-stage fail-closed construction is sound in outline — but it is not a new architecture family; it is a re-derivation of the already-registered NLB/1 design occupying the same FK-NLB-3 successor slot (ASM-0900), which already contains grammar-constrained decoding (as the secondary arm, ASM-0905(2)), trie-constrained entity emission, calibrated abstention with a risk-coverage deliverable, the SmolLM2-class shortlist, and the engine-validation backstop. Where H-GWP differs from NLB/1 it is mostly weaker in exactly the places NLB/1's review cycle hardened: legacy K=1 corpora (pre-declared non-decisive, ASM-0944) instead of NLB-EVAL/1 K=3 with five-role separation; 60-item DEV tau (unpowered against the 2% dangerous-wrong bound) instead of CAL-A/CAL-B splits; no acceptance layer for well-typed inversions; no product-automaton mask-equivalence check. Its genuinely novel deltas are small but real: the in-grammar ABSTAIN production as a load-bearing S1 mechanism, prompted/training-free operation (which, if it worked, would beat NLB-FE/1's fine-tuning recipe on cost and bytes), and dynamic type-conditioned relation masking (partially anticipated by ASM-0947's per-family type constraints, and by upstream item 3). Verdict: QUEUE — fold those deltas into NLB/1 as pre-registered arms rather than develop a parallel family; developing it standalone would fork a frozen instrument and invert the ladder. The cheapest experiment as proposed is NOT decisive: by the programme's own registered governance a pass on the legacy corpora cannot clear G-NLB and therefore cannot unblock the store-touching families, which is the proposal's entire option-value argument. However, a version IS GPU-runnable now within governance: run the prompted constrained-decoding arm as an NLB-0-B-class screening diagnostic (ASM-0904/0944 semantics: explicitly non-decisive, pre-frozen operating point, no cycle consumed) on the l3a vertical only (a5 is design-blocked per ASM-1342), SmolLM2-135M/1.7B-quantized plus the never-gated Qwen2.5-7B ceiling, using the existing poc/modal/modal_g8_vllm.py vLLM scaffold and the committed data/nlb-phrasings-l3a corpus — buildable in under a day, fills the idle Modal account, and answers the one question that steers real money: does prompted CD reach the 0.80-retention/0.01-dangerous-wrong headroom band, i.e., is NLB-FE/1's fine-tuning leg necessary at all. The upstream sparq items are correctly identified as non-blocking and correctly gated behind a PASS.
