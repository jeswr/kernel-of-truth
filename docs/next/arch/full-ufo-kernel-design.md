## Architecture

Six-layer design, obeying the EXPR-1 ladder (docs/next/arch/expressivity-analysis.md: cheapest layer that suffices; every outer-layer rule carries a gap code + axis). Epistemic status: STIPULATED architecture assembled over MEASURED code-state anchors, one ARCHITECTURE-VERDICT (ufo-rdf12-expressibility.md YES-WITH-EXTENSION), and LIT-BACKED UFO theory (Guizzardi 2005); no thesis feasibility conclusion anywhere.

L0 — kernel (unchanged). NSM profile-1 explications + content-addressed concept IDs remain the SOLE meaning layer (docs/kernel-design-directives.md §1). Full UFO is a formal-ontological SIDECAR, never a replacement formalism.

L1 — UFO commitment layer. Per-concept authored kot-ont-bridge/1 records (nsm-ufo-bridge.md B1, default `underdetermined`) carry FULL-UFO classifications: meta-category (endurant sortal {Kind, Subkind, Role, Phase} / non-sortal {Category, Mixin, RoleMixin} / perdurant-event / trope-moment {Quality, Mode-Disposition, Relator}); rigidity flag (rigid / anti-rigid / semi-rigid); identity provider (which Kind supplies identity to each sortal); dependence edges (inherence moment→bearer, mediation relator→mediated, participation endurant→event). Carrier: gUFO IRIs where they exist + the shipped ufo-rdf12/0.1 extension vocabulary (holdsIn / falseIn / existsIn / accessible / closedFor / identityCriterion / sameContinuant). Never inferred from prime occurrence — the KIND≠ufo:Kind false-friend rule is enforced.

L2 — world/claim layer (RDF 1.2). Situations/worlds are first-class nodes; claims are quoted triple terms <<( s p o )>> attached via rdf:reifies reifiers carrying ufo:holdsIn / falseIn / provenance / attitude edges. Quoting never asserts (opacity conformance-tested). This is the load-bearing move: it turns UFO's modal content into DATA, so rigidity/dependence/identity become ordinary safe Horn rules over (world, proposition) pairs instead of unencodable modal logic.

L3 — checking engine (sparq-reason + differential twin), the UFO-SN3 profile in three ordered regimes: (1) OWL-RL closure (existing, conformance-tested); (2) `ufo-sn3` materialisation — positive, function-free, range-restricted, situation-scoped rules: rigidity propagation across supplied accessible worlds while the individual exists; existential-dependence existence propagation; relator mediation → derived material relations; range-restricted identity criteria → sameContinuant merges; disposition trigger→activation; (3) closed-scope VALIDATION — stratified missing-witness / partition / cardinality checks only inside declared closedFor scopes (anti-rigidity counterworld witness, phase-partition completeness, relator-participant presence), emitting violations never entailments. New regime tag `ufo-sn3` extends {owl-rl, horn-def, policy}; every derivation proof-carrying and fail-closed. Ladder justification: GAP-MODAL is axis-1 for OWL (PROPOSED-ASM-1237) but resolved IN-STACK at L-B by reifying worlds as data; GAP-NAF for witness checks is axis-3 and is routed to the validation stratum, never entailment.

L4 — UFO-grounded vector kernel (sparq-vectors), house pattern throughout (read-only prior reader → encoder/sampler consumer behind an on/off ablation → paired multi-seed ComplEx adoption gate): (i) rigidity-split type encoding — rigid-kind membership as a stable identity block, role/phase membership contextual/situation-conditioned; (ii) UFO-DERIVED disjointness into DisjointnessOracle — all distinct Kinds pairwise disjoint by the unique-ultimate-sortal theorem, plus the rigid-cannot-specialise-anti-rigid theorem as a subsumption mask, both answer-safe, zero owl:disjointWith authoring; (iii) identity-criteria mask — sameContinuant/entity-resolution candidates require a shared identity provider (hard mask, DisjointnessOracle pattern); (iv) relator/qua composition prior — material-relation scores tied to mediating relator rows (n-ary structure the binary KGE currently fakes); (v) trope-inherence prior — moment rows anchored near bearer rows; endurant/perdurant/trope meta-category sub-block with hard disjointness; (vi) statement rows for quoted triples via a compositional encoder over child ids, making annotation/world edges trainable at all.

L5 — host coupling: unchanged programme-3 seams — H-VL verify-retry with the UFO-SN3 checker as the deductive gate (verify.rs propose-then-verify, fail-closed) and H-RULE-CD engine-derived continuation sets; oracle-diagnostic labelling and NLB gating inherited (ASM-0814).

CONTRAST — what full UFO buys that gUFO cannot. gUFO-in-OWL keeps the taxonomy and loses the theorems (expressivity-analysis §2.6 / PROPOSED-ASM-1237): the current stack can only ever use gufo:Kind/Role as inert class labels plus asserted owl:disjointWith, which is exactly why taxonomy.rs:33 scoped the gufo prior as a deferred annotation-reading tweak and eval.rs's gufo_prior axis is hard-wired false. Full UFO adds, concretely: (1) EXECUTABLE MODAL CONTENT — world-indexed rigidity checking, anti-rigidity witness validation, existence propagation: statable nowhere in the gUFO/OWL rendering, executable here as Horn over quoted triples; (2) DERIVED CONSTRAINT MASS — Kind-pairwise disjointness + rigid⋢anti-rigid as theorems give the oracle dense repulsion/mask structure with zero authoring, where gUFO yields only what someone asserted; (3) IDENTITY CRITERIA — gUFO has no executable identity semantics; full UFO's identity-provider discipline yields an answer-safe entity-resolution mask + sameContinuant rules; (4) RELATORS/QUA — gUFO names Relator; full UFO's mediation/dependence/derivation axioms are the usable content; (5) TROPE/ENDURANT/PERDURANT discipline as geometry, not labels; (6) CLAIMS-ABOUT-CLAIMS — the gUFO direction has no statement layer; quoted triples let worlds/beliefs/negative facts exist without assertion. What full UFO does NOT buy: decision procedures for unrestricted modal/deontic validity, arbitrary identity formulae, or unrestricted mereology (UFO-SN3 verdict boundaries), and no typing-soundness improvement by itself — the g2 lesson is that discipline, not vocabulary, is what helps.

## Grounding mechanism

Three coupled grounds, each with an explicit non-claim. (1) MEANING GROUNDING: NSM explications remain the sole meaning layer; every UFO commitment is an authored, endorsed kot-ont-bridge/1 judgment with `underdetermined` as the default outcome — UFO categories are never mechanically read off primes (nsm-ufo-bridge.md §6.2: prime occurrence → candidate reading → authored judgment → validation → executable axiom, never prime → axiom). (2) DEDUCTIVE GROUNDING: closure-before-vectorise (structure.rs close_for_vectorise) is extended so every vector-side prior consumes only triples the proof-carrying closure materialised (OWL-RL + ufo-sn3 regimes), and every vector proposal is re-checked by the same engine before becoming an answer (verify.rs propose-then-verify, fail-closed; masks are answer-safe — they remove only provably-wrong candidates). Derivations carry regime tags + proof trees end-to-end, so a UFO-grounded answer is auditable to its endorsed bridge records and world facts. (3) CLAIM/PROVENANCE GROUNDING: RDF 1.2 quoted triples + rdf:reifies let the world model hold claims, beliefs, counterfactual situations, and explicit negative facts WITHOUT asserting them; reifiers carry holdsIn/provenance/attitude; opacity (quoting never asserts) is conformance-tested, and the existing PROV-O weight-mining pattern (provenance.rs) extends to reifier-carried confidence. Explicit non-claim, carried from nsm-ufo-bridge §7.6: none of this transfers human/sensorimotor grounding into the artifact; the claim is auditability, reference fixity, and proof-carrying checkability only.
