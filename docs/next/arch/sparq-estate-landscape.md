## 1. sparq-vectors — vectorisation estate (agent `sparq-vectors`)

**Summary.** sparq's vectorisation estate (crates/sparq-vectors, ~30k LoC) is a layered, feature-gated system. Base layer (always on): a per-dict-term-id f32 VectorStore (`.spqv` mmap, store.rs) with exact/DiskANN/HNSW search (ann.rs, diskann.rs), out-of-process embeddings via an Embedder trait (embed.rs), entity verbalisation to text (verbalize.rs), and a v3 embedding-provenance header (spqv_provenance.rs). Structure-aware layer (`structure` feature, epic sq-0wo9e): P0 closure-before-vectorise (sparq-reason RDFS/OWL-RL materialisation) + type-constrained (domain/range) negative sampling (structure.rs); P1/P2 typed-literal sub-vector Blocks with per-block metric tags — order-preserving numeric, boolean, date, enum codebook, QUDT units, SHACL/OWL prior reader (encode.rs, codebook.rs, units.rs, shacl_priors.rs); P3 rdfs:subClassOf taxonomy block (Euclidean default, hyperbolic only past a measured-distortion gate) + owl:disjointWith repulsion/hard-mask (taxonomy.rs); P4 grounding dispatcher (grounding.rs); P5 neuro-symbolic propose-then-verify (verify.rs). PROV-O handling is NOT an embedding: provenance.rs mines a per-triple weight w(t)∈(0,1] from pkg:confidence/pkg:assurance/prov:wasDerivedFrom and threads it into the training loss (CKRL-style down-weighting) behind a WeightMode on/off ablation. KGE layer (`kge` feature): a hand-rolled CPU DistMult/ComplEx trainer over [Id;3] dict-id triples (train.rs) and a filtered link-prediction harness (eval.rs) with a 2x2 closure-x-type-negatives ablation, paired multi-seed statistics, long-tail breakdown, and a synthetic gUFO slice (gufo:Kind/Role/Phase meta-typing where the rigid kind must be DERIVED by closure). Crucially for KoT: (1) RDF 1.2 triple terms already exist end-to-end in the substrate — sparq-core's dict has TermParts::Triple([Id;3]), the parser accepts <<( s p o )>> / rdf:reifies, and fingerprint.rs recursively folds nested triple terms (tested at fingerprint.rs:519) — but every vectorisation layer above DROPS them: is_entity() in train.rs/eval.rs/structure.rs matches only Iri|Blank so triples touching a triple term are silently excluded from KGE positives, grounding.rs:793 renders a triple term as the empty string, and rewrite.rs:649 excludes them from vec: predicate bindings. So a quoted-triple-bearing graph loads, stores, and fingerprints fine, but its quoted-triple structure is invisible to the embeddings. (2) UFO-awareness is deliberately stubbed: taxonomy.rs:33 states the gufo:Kind/Role rigidity split is 'the design's optional/last prior... deferred (a tracked bead) and not implemented'; eval.rs carries a gufo_prior ablation axis on AblationCell that is hard-wired false (a no-op placeholder awaiting the prior), and the synthetic gUFO slice already exercises rigid-kind derivation. The house pattern for adding both is well established: a small read-only id-scan 'prior reader' module (shacl_priors.rs and provenance.rs are the two existing exemplars), an encoder/sampler consuming it behind an on/off ablation switch, and adoption gated on measured paired multi-seed lift under the asymmetric (ComplEx) model.

### Key capabilities

- Per-term-id mmap vector store (.spqv) with exact, DiskANN/Vamana, and opt-in HNSW search; delta sidecar; quantization; hybrid RRF fusion; graph-staleness fingerprint guard that ALREADY folds RDF 1.2 triple terms recursively (fingerprint.rs)
- Closure-before-vectorise: sparq-reason RDFS/OWL-RL materialisation so entailed type/subClassOf/domain/range facts are real triples before any encoder or sampler reads them (structure.rs close_for_vectorise)
- Type-constrained negative sampling (Krompass 2015) from declared + observed domain/range via sparq-introspect, fail-open to uniform, behind a SamplingMode on/off ablation
- Per-vocabulary/typed sub-vector Blocks with per-block metric tags and a metric-correctness guard: order-preserving numeric, boolean, date encoders, enum Codebook, QUDT unit normaliser, SchemaHeader (encode.rs/codebook.rs/units.rs); SHACL/OWL prior reader (shacl_priors.rs)
- Taxonomy block over the rdfs:subClassOf DAG (Euclidean default; hyperbolic only past a measured-distortion GeometryGate) plus an answer-safe DisjointnessOracle (train-time repulsion pairs + serve-time hard mask) (taxonomy.rs)
- PROV-O/DQV provenance-quality weighting w(t) mined from pkg:confidence/pkg:assurance/prov:wasDerivedFrom, threaded into KGE loss, pooling, and query-time fusion weights behind a WeightMode ablation (provenance.rs)
- In-tree shallow KGE: DistMult + ComplEx trainer (deterministic CPU SGD over dict-id [Id;3] triples, train.rs) and a filtered link-prediction harness (Bordes protocol, leakage-asserted splits, 2x2 ablation matrix, paired multi-seed deltas, long-tail breakdown) (eval.rs)
- A synthetic gUFO slice generator (gufo:Kind/Role/Phase meta-typing, rigid kind derivable only via closure, community-structured learnable edges) plus a gufo_prior ablation axis already plumbed through AblationCell — currently a hard-coded no-op awaiting the actual prior (eval.rs:298,543,986-1148)
- Neuro-symbolic propose(vector-ANN)-then-verify(SHACL + OWL-consistency deductive gate) pipeline, fail-closed (verify.rs)
- Verbalisation/grounding to text for text-embedding pipelines, RDF 1.2 lang--dir literal aware (verbalize.rs, grounding.rs)

### Gaps

- Quoted triples/triple terms are load-able but embedding-invisible: is_entity() at train.rs:353, eval.rs:116, and structure.rs:422 matches only TermParts::Iri|Blank, so collect_positives (train.rs:375) and Splits (eval.rs:193) silently DROP every triple whose subject/object is a triple term — rdf:reifies annotation edges never train and triple-term ids never get embedding rows
- No compositional encoder for a triple term: nothing builds a statement embedding from the child (s,p,o) ids, even though the recursive-resolution template already exists in fingerprint.rs:207-300 and the store is keyed by dict id (a triple term id could hold a row today)
- Triple terms verbalize to nothing: grounding.rs:793 maps TermParts::Triple(_) to the empty string, and verbalize.rs handles only Iri/Blank — the text-embedding path loses all quoted-triple content; rewrite.rs:649 also excludes triple terms from vec: predicate VALUES bindings
- No eval protocol or synthetic slice for statements-about-statements (no RDF 1.2 analogue of synthetic_gufo_ttl); no negative-sampling semantics for corrupting inner vs outer positions of a quoted triple
- UFO/gUFO prior explicitly deferred: taxonomy.rs:33 defers the gufo:Kind/Role rigidity split; there is no gufo prior reader (no rigidity classification, no identity-providing-kind extraction, no relator/gufo:mediates handling, no endurant/perdurant/trope distinction anywhere in the crate)
- eval.rs AblationCell.gufo_prior is always false — the ablation slot exists but nothing implements the arm (eval.rs:298-301, 543)
- The DisjointnessOracle reads only owl:disjointWith/AllDisjointClasses/complementOf — it does not exploit UFO-provable disjointness (endurant vs perdurant, distinct identity-providing kinds), and the NegativeSampler does not exploit rigidity (wrong-rigid-kind corruptions are guaranteed negatives, a strictly stronger prior than domain/range)
- Trainer is research-grade CPU-only (no GPU/minibatch); DistMult symmetric so all verdicts must be read off ComplEx — fine for measurement, not production KGE

## 2. sparq-reason + term model — reasoning estate (agent `sparq-reason`)

**Summary.** sparq ALREADY has a first-class RDF 1.2 term model for quoted triples: triple terms are a native dict term kind (`Stored::Triple([Id;3])` / `TermParts::Triple`), structurally interned via `intern_triple_ids`, hashed, spill-serialized, compaction-remapped, and totally ordered (component-wise) in the reasoner-side comparator. The Turtle and N-Triples/N-Quads parsers in sparq-core implement the full RDF 1.2 surface — triple terms `<<( s p o )>>` (object-position only), reified triples `<< s p o >>` desugared to `R rdf:reifies <<(s p o)>>` byte-identical to oxttl, reifiers `~ id?`, annotation blocks `{| … |}`, 128-deep nesting guard. NOTE: sparq-parse is NOT a parser — it is a zstd/gzip response-compression crate; RDF parsing lives in sparq-core (ttl.rs/nt.rs) and SPARQL 1.2 (quoted-triple patterns, triple-term JSON results) is already wired through sparq-engine with the sparql-12 feature always on. Reasoning profiles: sparq-reason = RDFS (rdfs2/3/5/7/9/11 semi-naive fixpoint over [Id;3]), OWL 2 RL subset (union-find sameAs entity rewriting), D-entailment (feature d-entail), N3 forward/backward chainer with quoted FORMULAE (an N3-native structure, distinct from dict triple terms), RIF-Core front-end, and an opt-in stratified Datalog dialect (NAF + COUNT, RDFox-parity track); sparq-reason-el = OWL 2 EL+⊥ classifier (CR1–CR9, ABox realization); sparq-reason-ql = OWL 2 QL / DL-Lite_R PerfectRef+tree-witness UCQ rewriter (CQ/UCQ gates always on, rewriting experimental); sparq-reason-dl = OWL 2 Direct-Semantics ALCH structural extract → profile check → tableau → fragment dispatch. Current triple-term behavior in reasoners: all rule engines join over dict ids, so a triple-term id flows through OPAQUELY — inference over quoted triples as opaque constants (typing reifiers, subPropertyOf over rdf:reifies, datalog rules binding a variable to a triple-term id) works TODAY with zero changes, and the union-find sameAs rewriting correctly does NOT substitute inside triple-term components (matching RDF 1.2 opaque semantics). What is missing for real inference over quoted triples: (1) no rule engine can DECOMPOSE or CONSTRUCT triple terms — the Datalog dialect grammar only allows IRIs/literals/variables in atom s/o positions, N3 quoted formulae have no dict representation, no rdf:reifies-aware builtins; (2) no position-validity guard — rdfs3 (range) derives `(o rdf:type C)` and prp-fp derives `(y1 sameAs y2)`, which put a triple-term id in SUBJECT position (ill-formed RDF 1.2) when the object is a triple term; (3) EL/QL/DL are fail-closed and explicitly exclude triple terms (EL's is_individual rejects them), which is correct — DL-side RDF-star has no standard semantics. Recommended vehicle: extend the Datalog dialect (the RDFox-parity path — RDFox itself does RDF-star this way): add `<<(?s ?p ?o)>>` patterns to the parser, unify pattern components against TermParts::Triple during joins, and intern derived triple terms in heads via intern_triple_ids. Effort M; RDFS/OWL-RL position guards S; N3 bridging L; DL-profile RDF-star XL/research.

### Gaps

- No rule engine can decompose or construct triple terms: the Datalog dialect atom grammar (datalog/parser.rs) only admits IRIs/literals/variables in s/o positions — no <<(?s ?p ?o)>> patterns, no head-side triple-term construction
- No position-validity guard in the materializers: rdfs3 (range) derives (o rdf:type C) and prp-fp/prp-ifp derive (y1 sameAs y2) — when o/y1 is a triple-term id these place a triple term in subject position, ill-formed under RDF 1.2 (grep confirms zero triple-term handling in rdfs.rs, owl.rs, datalog/)
- No rdf:reifies-aware rules or builtins anywhere in the reasoning estate (no reifies vocab constant in any reason crate) — e.g. no assertedness bridge (r rdf:reifies <<(s p o)>> ⊢ (s p o)) even as an opt-in policy rule
- N3 quoted formulae are a separate N3-native structure with no dictionary representation (n3/compiled.rs) — they are NOT RDF 1.2 triple terms, and no bridge exists between the two
- EL/QL/DL extract layers fail-closed on triple terms (EL is_individual explicitly excludes them, extract.rs:882) — correct behavior, but it means the DL-profile stack is not a vehicle for quoted-triple inference and no standard OWL semantics exists to target
- sparq-parse contains no term model at all — it is a zstd/gzip compression codec crate; the task's assumption that it holds parser/term code is wrong (RDF parsing is in sparq-core ttl.rs/nt.rs)

### Effort to extend

M — The recommended real extension, in the Datalog dialect (feature `datalog`, the RDFox-parity track, mirroring how RDFox does RDF-star): (a) parser.rs: admit <<( s p o )>> triple-term patterns (constants and variable components) in atom subject/object positions; (b) eval.rs: when a triple-term pattern meets a bound id, check TermParts::Triple and unify component ids (recursive, bounded by the existing 128-depth invariant); (c) head emission: dict.intern_triple_ids for constructed triple terms; (d) stratify.rs untouched (predicate-level dependency graph unaffected). L — N3 bridging (map dict triple terms into the N3 term universe or vice versa): new N3 term kind + unification semantics; not needed for the Datalog path. XL — EL/QL/DL profile-level quoted-triple reasoning: no standard OWL 2 RDF-star semantics exists; keep the fail-closed rejections.
