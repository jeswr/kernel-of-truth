## 0. Verdict in three lines

1. **Continuation sets: the engine does not emit them — measured no as an
   interface; measured yes as a substrate.** No continuation, enumeration, or
   trace API exists anywhere on the engine (mechanised inspection found zero
   matching attributes or defs), but a 90-line external adapter over six
   public engine attributes derives EXACT admissible-continuation sets at
   28 µs–368 ms per partial query on the pinned stores (the 368 ms extreme is
   the 72,839-candidate define enumeration) [MEASURED: probe legs
   0+3, sha256 6990c047baa3845d].
2. **Derivation traces: partially emitted.** Every accept carries a
   two-field certificate (provenance world-record ids + license axiom refs);
   replaying it on a store restricted to exactly the cited records reproduces
   the answer for **464/600** covered l3a answers and **438/855** covered a5
   answers — and systematically fails for the rest (the certificate omits the
   licensing-support class assertions and the class-licensing axioms it
   consumed). Refusals emit **no** certificate at all (`{status, code,
   reason}` prose only) [MEASURED: probe leg 2, sha256 6990c047baa3845d].
3. **Consequence: the NTP §8 gate is NOT cleared today, and clearing it is a
   bounded, priced, CPU-only build** (§5: ~330–550 additive engine/test
   lines, zero GPU, byte-identical default emissions) with a mechanical
   acceptance test — the probe's replay leg flipping to 100% closure. No
   trace- or mask-consuming arm should be budgeted before that lands
   (ASM-0981, ASM-0982).

## 1. What was probed, and why this operationalisation

- PREMISE: the engine under test is the unmodified production engine
  `tools/axiom/kot_axiom.py` (five ops: unique/lookup/count/instance/define;
  fail-closed ERR_* refusals; CWA store validation) plus the code-vertical
  oracle `tools/axiom/kot_code.py`, built exactly as the frozen l3a/a5
  instruments build them, over the pinned corpora `data/axioms-v0` +
  `data/world-v0`, `data/code-axioms-v0` + `data/code-world-v0`, and the
  definitional substrate (`data/axioms-definitional-v0` + onto-obo shards +
  mint bridge) [MEASURED: poc/ntp-gate/probe.py build calls; output sha256
  6990c047baa3845d].
- PREMISE: "continuation set" and "replayable derivation trace" are
  operationalised at the consuming-arm interface — G1/G2 per-field
  admissible-value sets for partial queries, and replay-closure of the
  emitted certificate, respectively [STIPULATED: ASM-0980].
- The probe (`poc/ntp-gate/probe.py`, stdlib-only, RNG-free, doubled-pass
  byte-identical) has four legs: (0) mechanised API-absence inspection,
  (1) emission inventory over the full pinned eval sets (900 l3a + 977 a5
  queries), (2) certificate replay for every covered answer, (3) worked
  continuation partials on both verticals, (4) trace-gap demonstrations.
  All outputs land in `poc/ntp-gate/out/`.

## 2. Measured findings

### 2.1 Emission inventory — what the engine emits today [MEASURED: sha256 6990c047baa3845d, leg 1]

- Answers are exactly `{status, value, provenance, license}`; refusals are
  exactly `{status, code, reason}` — 1455/1455 answers and 422/422 refusals
  across both eval sets, no other shape observed. The doubled pass is
  byte-identical on both verticals.
- The define-op emits the same certificate shape (provenance = onto-obo
  record id, license = endorsement constraint ref) and its three refusal
  modes fire as specified (`ERR_DEFN_UNRESOLVED`, `ERR_NO_DEFINITION`
  observed; index sizes: 72,839 licensed / 13,824 resolved / 3,387
  unresolved).
- Raw latency context, this box (2 shared cores, single-thread CPython, per
  query incl. Python dispatch): l3a mean 6.3 µs, p95 10.5 µs; a5 mean
  8.7 µs, p95 12.2 µs. Context only, no verdict; consistent with the
  programme's µs-engine premise as restated in NTP.md §6.

### 2.2 Certificate replay — is the emission a closed derivation trace? [MEASURED: sha256 6990c047baa3845d, leg 2]

Replay = rebuild a store from ONLY the records the answer names (provenance
world ids + license axiom files), re-run the same query, require the same
value (ASM-0980 clause 2).

| vertical | covered answers | replay exact | replay fails | failing families (all of the family, refusal observed) |
|---|---|---|---|---|
| l3a family/world | 600 | **464 (77.3%)** | 136 | count-maker 43/43 `ERR_UNLICENSED_COUNT`; unique-maker 43/43 `ERR_UNLICENSED_UNIQUE`; instance-true 50/50 `ERR_TERM_UNLICENSED` |
| a5 code | 855 | **438 (51.2%)** | 417 | where-defined 201/201 `ERR_UNLICENSED_UNIQUE`; instance-true 216/216 `ERR_TERM_UNLICENSED` |
| define (spot, 3 subjects) | 3 | 3 | 0 | — (but see bridge caveat below) |

The failures are systematic, not noisy, and each names a missing record
class the derivation consumed but the emission does not cite:

1. **Licensing-support class assertions.** Cardinality-licensed unique/count
   answers require `subject ∈ classes[subj_class]`; that class-assertion
   world record is consulted but never enters `provenance` (families
   count-maker, unique-maker, where-defined — 287 answers).
2. **Class-licensing axioms.** `instance` TRUE answers emit
   `license: ["asserted"]`, citing no axiom file at all, although the
   concept's admission to `licensed_classes` (via some disjoint/range/domain/
   cardinality axiom) was a precondition — replay refuses
   `ERR_TERM_UNLICENSED` (266 answers).
3. **Refusals carry nothing.** A refusal names a code and prose; it cites no
   record, no violation object, no terminating step — there is no reject
   trace to replay.
4. **Define caveat:** the 3/3 define replays consumed the pinned mint bridge
   from the corpus, which the certificate does not name — the definitional
   certificate is closed only modulo that pinned artifact.

### 2.3 Continuation sets — no API, exact sets derivable [MEASURED: sha256 6990c047baa3845d, legs 0+3]

- Mechanised inspection: zero attributes on `Engine` and zero function defs
  in `kot_axiom.py` match
  `continu|legal|admiss|enum|next_|trace|proof|deriv|expla` — the engine has
  no continuation/trace surface of any kind. `query()` is all-or-nothing.
- The probe's `ContinuationAdapter` (~90 lines, consuming only the public
  attributes `licensed_rels`, `licensed_classes`, `entities`,
  `defn_licensed`, `defn`, `query`) derives exact G1 (licensed-vocabulary)
  and G2 (answer-admissible) sets. Worked partials:

| partial | next field | G1 | G2 | engine calls | wall |
|---|---|---|---|---|---|
| `{}` (which ops answerable) | op | 5 | 5 | 9,229 | 102 ms |
| `{op:unique}` | rel | 5 | 3 | 1,771 | 9.0 ms |
| `{op:unique, rel:mother, dir:fwd}` | subject | 324 | 122 | 324 | 2.0 ms |
| `{op:instance, entity:elvis}` | concept | 3 | 2 | 3 | 28 µs |
| `{op:count, rel:maker, dir:inv}` | subject | 324 | 43 | 1,167 | 6.9 ms |
| `{op:define}` | subject | 72,839 | 13,824 | 72,839 | 368 ms |
| code `{op:callers-of}` | of | 222 | 74 | 222 | 1.4 ms |

- Validity cross-check: the adapter's G2 sets exactly recover the pinned
  eval generators' coverage counts (122 unique-mother, 43 count-maker, 74
  callers-of covered queries) — the two independently-built enumerations
  agree.
- The G1→G2 gap (324→122; 222→74) is precisely the mask-precision delta
  H-RULE-CD's engine-derived masks would deliver over vocabulary-only
  masking. [EXTRAPOLATION — sets of this shape being useful as decode-time
  masks at model-vocabulary granularity is the P3-D-SSD design question
  (NTP.md §7-Q4), not shown here; these costs are store-scale-specific and
  the brute-force strategy is exponential in remaining fields in general,
  which is why B2 mandates index-backed derivation.]

### 2.4 Trace gaps — computed internally, never emitted [MEASURED: sha256 6990c047baa3845d, leg 4]

- The CWA pass builds structured violation objects
  `{code, detail, implicated}` (6 in the l3a store, 3 planted in a5), but an
  `ERR_CONFLICT` refusal names none of them: the probe shows the refusal for
  query q0831 side-by-side with the exactly-matching in-memory
  `VIOLATION_FUNCTIONAL` object it will not cite.
- inverseOf canonicalisation flips are invisible: q0461 answers a `lookup`
  through the canonical-partner flip and nothing in the emission records
  that the store direction consulted was the reverse of the queried one.
- Refusals do not name the terminating step of the pre-registered validation
  order (shape → term licensing → entity → conflict → op license → records),
  although the code path is deterministic — the code-to-step mapping exists
  in source only.

## 3. Answer to the NTP §8 checklist, item by item

NTP.md §8 requires demonstrating the executor "can actually EMIT what each
proposed arm consumes — proof states, legal-action/continuation sets, step
traces, compiler-style diagnostics, and search transitions."

| §8 item | today | grade |
|---|---|---|
| legal-action / continuation sets | no API; exact sets derivable externally at measured µs–ms cost | **ABSENT as emission; substrate shown** |
| step traces (accepts) | provenance+license certificate; closed for 77.3% (l3a) / 51.2% (a5) of covered answers | **PARTIAL** |
| step traces (rejects) | code + prose only; nothing replayable | **ABSENT** |
| compiler-style diagnostics | closed ERR_* code set (machine-matchable, the a5 control families score on exact codes) but no structured operands (no step, no violation refs, no expected/got) | **PARTIAL** |
| proof states / search transitions | the v0 engine is a depth-1 deterministic lookup oracle with a pre-registered validation order — there IS no multi-step search; its complete derivation is the validation-step + consulted-records sequence, which B1 emits. Anything deeper (subClassOf closure) is refused at load by design (`ERR_AXIOM_UNIMPLEMENTED`) | **collapses to B1; deeper states out of scope** (ASM-0980) |

- DECISION: the oracle/API gate is **not cleared** by the current engine;
  the AlphaGeometry-analog and Discrete-NAR state-supervision mappings in
  NTP.md (§3, §4.3) therefore remain STIPULATED gaps and MUST NOT be cited
  as available capability until §5's build passes its acceptance tests
  [STIPULATED: ASM-0981; baseline numbers MEASURED per §2, sha256
  6990c047baa3845d].

## 4. Why this is the right bar (and not a stricter one)

- PREMISE: the dependent arms consume, at minimum: admissible-token sets
  (P3 masks), self-contained derivations for curricula and audit (P6, H-VL),
  and machine-readable failure operands for retry training (H-VL, RLEF-shape
  feedback) [STIPULATED: ASM-0980; the arm-side requirements restate
  round1-critique-synthesis A.8/A.9 and NTP.md §5–6].
- A stricter bar (theorem-prover-grade proof objects, search-tree
  transitions) tests capability no registered arm consumes and the v0 engine
  was never specified to have; a weaker bar ("the engine answers and
  refuses deterministically") is what l3a/a5 already measured and is exactly
  the assumption NTP.md flagged as insufficient. Replay-closure is the
  weakest property that makes a trace *checkable by reconstruction* — the
  property distillation curricula and audit both need.

## 5. The build: what closes the gate, priced [STIPULATED: ASM-0982]

Three additive, opt-in, CPU-only engine extensions; the default `query()`
emission stays byte-identical (the frozen l3a/a5 instruments consume it):

- **B1 — trace emission** (`query(q, trace=True)`): ordered validation-step
  list; per step the outcome and EVERY consulted record reference — axiom
  constraint refs, world-record ids *including licensing-support class
  assertions* (§2.2 items 1–2), canonicalisation flips (§2.4), definitional
  provenance + consulted mint-bridge entries (§2.2 item 4). Emitted for
  accepts AND rejects. ~150–250 lines + tests.
- **B2 — continuation API** (`continuations(partial)`): the ASM-0980 G1/G2
  sets, derived analytically from the engine's existing indexes (`rel_fwd`
  keys, `classes`, `defn`, licensing sets) rather than by query loops; the
  probe's brute-force adapter is the frozen reference semantics, equivalence-
  tested. ~120–200 lines + tests.
- **B3 — structured refusal diagnostics**: refusals gain machine operands —
  terminating step, and for `ERR_CONFLICT` stable references into the
  store-violation objects (§2.4). ~60–100 lines.

Acceptance, all mechanical, all required (ASM-0982): (a) replay closure
100% on both pinned eval sets consuming trace-named records (baseline:
464/600, 438/855); (b) B2 ≡ reference adapter on all worked partials;
(c) byte-identical default emissions pre/post (both frozen instruments);
(d) doubled-pass determinism on all new surfaces. Estimated total ~330–550
added lines, zero GPU, ≤2 focused agent-days. Then re-run the gate as the
registered P3-E-NTP-GATE-0 (ASM-0981) — its instrument is `probe.py`
re-pointed at trace-named record sets.
