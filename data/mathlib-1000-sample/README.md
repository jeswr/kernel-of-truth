# mathlib-1000-sample — the D-ML input corpus for experiment g8

**What this is:** a seed-0, deterministic, uniform-random sample of **1,000
Mathlib declarations** extracted as `lean-ref/1` formal-reference records. It is
the `mathlib-1000-sample` corpus pinned (as `PINNED-AT-INPUTS:d-ml`) by
`registry/experiments/g8.json` — the **null denominator** for g8's fragment-gate
primary endpoint (HS8 / NF3: Lean-minting viability).

Generated by the Opus execution role as g8 material-generation ops. **This
corpus is only the input material; it is not a g8 run and produces no verdict.**
The g8 instrument (fragment gate, forward map F, LLM locator, round-trip) that
consumes these records is Fable design work and is not built (see the parent
report / bead).

Companions: `docs/design-lean-route.md` (route survey; this corpus uses the §5.2
doc-gen4 fallback crawl), `docs/design-dl-from-nsm-and-lean-reconstruction.md`
§5.3 (the fragment-gate/forward-map/round-trip protocol g8 measures),
`data/math-lean-sample/` (the 70-record feasibility sample this scales up),
`docs/design-bulk-kernel.md` (provenance rule).

## Honesty contract

Records are **formal-reference records** (`"record":"lean-ref/1"`,
`"status":"formal-reference"`): annotation-layer pointers into Mathlib,
identity-anchored on **(source.mathlibCommit, name)**, minting **no**
`urn:concept:` ids. **No field is concept-hash-boundary content.** In
particular `signaturePretty` is the doc-gen4 pretty-printed signature — a
rendering that depends on notation, dot-notation, implicit-binder display,
auto-generated universe/instance names, and pretty-printer settings; the same
kernel declaration admits many such strings (design-lean-route §2). These
records exist to be the fragment-gate population and bridge/annotation material,
nothing more.

## Provenance & reproducibility

- **Enumeration index:** `declarations/declaration-data.bmp` (66 MB JSON served
  as `image/bmp`), sha256 `ea533afebfa00deb1d862411f59a28532692c10611f1725eab3ff04ad3451fc8`,
  daily doc-gen4 build last-modified `Fri, 10 Jul 2026 01:26:54 GMT`. The index
  itself is **not** byte-archived in-repo (size); its sha256 + the mathlib
  commit in every record's gh_link are the content pins. `sample.json` records
  the index sha and the exact selection.
- **Mathlib commit (content pin):** `e3b73828b03c9961c9b33aa95f1d2dc0dca82028`
  (single commit across the whole build; `extract.mjs` fails closed on any
  cross-module commit mismatch).
- **Sampling algorithm (seed 0):** population = every declaration-data.bmp entry
  whose docLink is under `./Mathlib/` (**315,970** Mathlib-proper declarations);
  each name N ranked by `sha256("g8:mathlib-1000-sample:seed=0\x00" + N)` hex
  ascending; first 1,000 taken. Fully reproducible from the index + `sample.json`.
- **Source archive:** `source-snapshot/*.html.gz` — the 866 module pages the
  records derive from, gzip-compressed (raw pages total 252.9 MB; the docs URLs
  rotate daily in place, so these are the only archival copy of the exact bytes).
  `fetch-log.json` records per-module raw-HTML sha256, bytes, and last-modified.
- **Determinism:** given byte-identical `source-snapshot/*.html.gz` + `sample.json`,
  `node extract.mjs` re-emits `records.jsonl` + `manifest.json` byte-identically
  (verified: two consecutive runs identical). No network in the deterministic phase.

## Extraction coverage (doc-gen4 signature-layer limitation)

Of the 1,000 sampled declarations, **966** render as top-level `decl` divs and
are extracted in full. **34** are auto-generated sub-declarations (structure
fields, constructors, class-projection theorems / `toXClass` instances) that
doc-gen4 does not render as their own top-level decl:

- `extraction: "sub-declaration"` (33) — present as a nested anchor inside a
  parent decl div; signature not extracted at the doc-gen4 layer.
- `extraction: "index-only"` (1, `AddMonoid.toZero`) — in the index but not
  rendered as an anchor on the page.

These 34 carry name/kind/module/docLink from the index with `signaturePretty:""`
and a `note`. All 34 are members of non-arithmetic structures (Topology /
Category / Algebra) and are out of the profile-M PA fragment regardless of
signature, so this limitation does not bias the fragment count. The
**ntp-toolkit primary route** (design-lean-route §5.1; bead
`kernel-of-truth-ncv`) is JSON-native and would extract all 34 in full.

## Files

| File | Role |
|---|---|
| `records.jsonl` | 1,000 `lean-ref/1` records, sorted by UTF-8 bytes of `name` |
| `sample.json` | the reproducible seed-0 selection + sampling algorithm + index provenance |
| `fetch-log.json` | per-module fetch provenance (URL, raw-HTML sha256, bytes, last-modified) |
| `manifest.json` | generated: commit, counts, kind/extraction distributions, per-snapshot gz sha256 |
| `extract.mjs` | deterministic extractor (zero-dep Node): `source-snapshot/*.html.gz` + `sample.json` → `records.jsonl` + `manifest.json` |
| `source-snapshot/*.html.gz` | the 866 gzipped doc-gen4 module pages (byte archive) |

Regenerate: `node extract.mjs` (no network, no deps).

## Record fields

`name`, `kind` (rendered doc-gen4 kind, e.g. `theorem`/`noncomputable def`),
`module`, `extraction` (`doc-gen4-decl` / `sub-declaration` / `index-only`),
`attributes`, `signaturePretty` (annotation only, non-canonical), `docstring`
(usually empty), `referencesPretty` (signature-level type dependencies —
hyperlinked constants; not proof/value deps), `hasEquations` /
`equationsUnrendered`, `source` (mathlib commit + file + line span, or docLink
for flagged records).
