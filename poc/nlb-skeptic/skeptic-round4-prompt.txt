BACKGROUND: this is the "NL-boundary" leg of two experiments, l3a-parse and
a5-nl. They test whether a deterministic (non-LLM) front-end parser can
recover a closed query grammar from blind, held-out natural-language phrasings
of two frozen evaluation sets (a family/world vertical and a code vertical),
compared against a measured gold-parse ceiling. A run is scored PASS or FAIL
by pre-registered Wilson-confidence-bound gates. The scoring code contains a
series of fail-closed VALIDITY CHECKS ("gates" G0-G7) whose job is to mark a
run INSTRUMENT-INVALID whenever the input data records are malformed or
internally inconsistent, so that a nominal PASS can never be reported off of
corrupt or contradictory measurements. Your review confirms those validity
checks correctly REJECT malformed or internally-inconsistent input records,
and correctly ACCEPT well-formed ones.

REVIEW HISTORY: the immediately preceding round of this audit (your own prior

found the round-2 remediation of the G0 supersede path and the instrument

  (FB-1) "G0 read-time defense accepts unreasoned and double-retirement

         reason, or one that retired an already-retired target, was
         classified `instrument_valid=True` in both scorers and both record
         orders — even though the official append path (log-append.py) had
         always rejected such a line. This contradicted the registered
         claim that the scorer forms an INDEPENDENT second enforcement of
         the same rule (defense in depth), not a mere pass-through.
  (FB-2) "Invalid boolean totals can crash instead of returning
         INSTRUMENT-INVALID" — the counts-integrity gate (G1) correctly
         detected a boolean-valued `n_control`/`n_covered` and set
         `g1_counts=False`, but the scorer then CONTINUED past the gate
         into Wilson/Holm arithmetic using the rejected value as a
         denominator. `n_control=True` crashed both l3a_parse.py and
         a5_nl.py with a `TypeError` (Python's `True > x` comparisons
         against a `complex` intermediate); a5_nl.py's `n_covered=True`
         reached the output with complex-valued Wilson bounds and then
         crashed on JSON serialization. Verdict generation treats a
         nonzero scorer exit code as `ERR_P2_ANALYSIS`, not the registered
         `INSTRUMENT-INVALID` verdict, so the pre-registration's promise
         that malformed counts are classified `INSTRUMENT-INVALID` was
         unreachable on exactly these inputs.
  (SF-3) "Freeze scanner still treats superseded ASM-0425 as a live
         citation" (should-fix) — `prereg-freeze.py`'s
         `scan_open_extrapolations` scanned every ASM token appearing
         anywhere in the pinned prereg document, including historical
         supersession prose, so a dry-run kept emitting a nonfatal pause
         flag for the already-superseded ASM-0425 even though the
         remediation claimed no live citation reached it.
