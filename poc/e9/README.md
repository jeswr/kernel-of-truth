## What this is, and what it is not (naming honesty — judgement call J1)

docs/poc-design.md pre-registers E9 as, verbatim:

> **E9 — decode-verify vs RAG (new, panel O5; phase 2).** A5's tier finally
> gets an experiment: factual-consistency benchmark, decode-verify vs vanilla
> RAG-with-citations, measuring what each catches that the other misses.
> **(rev 3, design-review change 3:) third arm = the deflationary null
> architecture — RAG over a hash-pinned definition-TEXT dictionary (no
> encoder, no structured records). The kernel machinery's marginal value over
> its own deflation is measured, never asserted.**

That phase-2 product experiment (decode-verify vs RAG) is **not** what this
directory runs, and it remains pre-registered and untouched. What this
directory runs — under the working label **E9-defl** — is the same
*deflationary principle* ("the kernel machinery's marginal value over its own
deflation is measured, never asserted", notes/panel-kernel-design-review.md
§3.1) applied to the **E5 nonce-transfer instrument**, which passed its
instrument gate 5/5 on 2026-07-07 (poc/e5, OUTCOME PASS). The question,
as directed:

> Is the kernel's signal specifically the (NSM-grounded) semantic content of
> the explications, or would ANY consistent structured deterministic vector
> set do?

E5's shuffled control ruled out "the vector *spectrum* alone suffices". It
did **not** rule out a subtler deflation: kernel vectors encode the
explication's *structure and size*, and the realizer glosses inherit
structure and size from the same AST — so part (or all) of E5's +0.285 could
travel through a purely structural channel (e.g. big-explication ↔ long-gloss)
with no semantics involved. E9-defl builds the strongest cheap version of
that deflation and measures the kernel's marginal value over it. A DEFLATED
outcome is a legal result of the programme, not an unthinkable (design-review
§3.1 verdict, applied one tier down).

## The deflationary control kernel (pinned construction — judgement calls J2/J3)

**defl-kernel:** for each of the 524 E5 concepts (poc/e5/inputs/
e5-concepts.json, reused byte-identically, sha-pinned), ONE scrambled
explication generated by the SAME seeded generator that produced the E4/E5
synthetic vocabulary (`generateExplication`, encoder synth v2), then encoded
by the SAME pinned encoder (kot-enc-B/1 @ D=8192, hash `40e8c8ba…`, fail
closed) through the SAME pinned X4 JL 8192→576, unit-normalised — i.e. the
identical machinery, fed semantically unrelated but **structurally matched**
explications:

- Seed scheme (pre-registered): seed = `e9/defl/<slug>` (+ `/retry<k>` on
  collision), size parameters matched to the concept's TRUE explication:
  - **synthetic concepts (470):** (topClauses, depth) copied EXACTLY from the
    E4 generator record — the generator's own parameters.
  - **authored concepts (54, J2):** the generator's parameters are not
    attested, so they are measured from the AST by a pinned proxy —
    topClauses = `clauses.length`, depth = maximum clause-nesting depth
    (clamped to the generator's 1–5 range) — recorded per concept in the
    artifact. Flagged: for authored concepts the match is
    proxy-parameter-level, not generator-exact.
  - **J3:** scrambles are PURE PRIME structures (the generator takes no
    concept pool — E4's own rationale), so authored concepts' scrambles do
    not reproduce their conceptRef structure; they match at the size level
    above. Flagged.
- Dedup, fail closed: a scramble's canonical-AST sha-256 must differ from
  ALL 524 true explications' shas and all other scrambles (labelled retries,
  recorded). Every scramble re-validated (`validateExplication`).
- What is preserved: deterministic, compositional, internally consistent
  code; same encoder; same spectrum statistics; same per-concept SIZE
  signature. What is destroyed: any relationship between a vector's content
  and its concept's meaning/gloss beyond size/shape.

**Everything else is the E5 instrument, byte-identical (J4):** same frozen
SmolLM2-135M, same adapter form/init/budget/LR rule, same training items,
same seen-validity items, same 24 nonces and 120 forced-choice eval items,
same glosses under the same published hash, same 5 paired seeds, same
scoring rubric. poc/e5 inputs are consumed READ-ONLY and sha-pinned in the
E9 manifest; poc/e5 code is imported READ-ONLY by the E9 runner (the
poc/e4-reuses-stats_e1 precedent).

## Arms (one fresh run — judgement calls J5/J6)

1. **true-kernel** — E5's kernel table (reused file, sha-pinned).
2. **shuffled-kernel** — E5's derangements (reused, per seed).
3. **defl-kernel** — the deflationary table above (same for all seeds;
   arms differ only in the frozen vector table, Common rule 1 pairing).

J5: E5's random arm is dropped (already characterized: chance on nonces).
J6: true and shuffled are RE-RUN fresh alongside defl in ONE run rather than
compared against E5's committed numbers — paired tests must not straddle the
GPU float non-reproducibility between runs. The rerun doubles as an E5
replication datum; drift vs the committed E5 numbers is reported
descriptively.

## Pre-registered statistics

- **Instrument-validity gate (unchanged from E5):** the TRUE arm must beat
  chance on the SEEN validity items in ≥4 of 5 seeds (per-seed one-sided
  exact binomial vs 0.2 over 500 items, p < 0.05); otherwise
  INSTRUMENT-INVALID, no claim in either direction.
- **Primary endpoint (one):** nonce slot-filling accuracy, **true-kernel vs
  defl-kernel** — per nonce j, d_j = mean over the 5 paired seeds of
  (acc_true[s,j] − acc_defl[s,j]); one-sided exact sign-flip permutation over
  the 24 nonce-level paired differences (statistic = Σ_j d_j; full 2^24
  enumeration, integer-lattice convolution; p includes the observed
  assignment), **α = 0.05**. This is the "marginal value over its own
  deflation", measured. Same pinned caveat as E5: nonces share the 5 trained
  adapter pairs; the seed-level secondary treats the training run as the unit.
- **Secondaries (Holm-corrected, m = 2):**
  - S1: **defl-kernel vs shuffled-kernel**, same nonce-level test, one-sided
    (defl > shuffled) — does the structural channel carry ANY transfer?
  - S2: seed-level one-sided exact paired sign-flip over the 5 per-seed mean
    nonce-accuracy differences, true vs defl (min attainable p = 1/32).
- **Outcome labels (pinned):**
  - gate failed ⇒ **INSTRUMENT-INVALID**.
  - primary rejects (true > defl, p < 0.05) ⇒ **PASS** — the kernel's signal
    is not exhausted by a structure-matched semantics-free code; marginal
    value of content shown on this instrument.
  - primary does not reject AND S1 rejects ⇒ **DEFLATED** — a
    semantics-scrambled structured code reproduces the transfer; E5's
    positive is re-attributed to the structural channel pending further
    evidence, and every external quote of E5 must carry this.
  - primary does not reject AND S1 does not reject ⇒ **AMBIGUOUS-NULL** —
    no measurable separation; reported as such (no TOST bound is
    pre-registered here; a power follow-up is filed instead).
  - All four are reported verbatim with full tables regardless.
- **Descriptives (no inferential claims):** recovered fraction
  (defl − shuffled)/(true − shuffled) per seed and pooled; drift of the
  rerun true/shuffled numbers vs poc/e5's committed run; defl seen-validity
  accuracy (a structure-only code should still interpolate seen concepts —
  its failure would be reported, and would itself be informative);
  compositional shared/novel splits; per-nonce and per-seed tables; margins.
- **No nonce exclusions; no post-hoc arms; ties score as incorrect** — all
  E5 pins inherited.

## Controls before real spend (pinned)

1. CPU end-to-end mock (tiny random-init model, 2 seeds, 3 arms) — mechanics.
2. **Planted-effect control:** synthetic per-item records with a known
   true-vs-defl effect fed through the VERBATIM stats path must yield PASS
   with the analytically exact p.
3. **Null control:** label-exchangeable synthetic records must not reject
   (fixed-seed check + analytic symmetric cases).
   All three run in `runner/check_smoke.py` and must be green before the
   Modal run; evidence committed.
