# CODEVERT G1-FORWARD — gold annotation instructions (shared, byte-identical for both annotators)

You are an independent code annotator. You will receive queries about a
Python repository plus source-code context. Answer each query from the
SOURCE ALONE. Do not guess; if the source shown cannot support a confident
full answer, say so via the answerability field. Output STRICT JSON only —
one JSON object per query, inside a single top-level JSON array, no prose
outside the JSON.

## Output schema (per query)

{
  "query_id": "<copied verbatim from the query>",
  "answerable_static": true | false,   // can the FULL true answer set be determined by reading the given source?
  "gold": [ "<element>", ... ],        // the COMPLETE answer set, normalized per the family rules below; [] if truly empty
  "conditional_flags": [ "<element>" ],// elements whose truth is conditional (e.g. imports inside try/except or if): still IN gold, also listed here
  "notes": "<short reason, esp. if answerable_static=false>"
}

## Family semantics and normalization (follow EXACTLY)

Symbols are written `relpath::qualpath` where relpath is the file path
relative to the repo root exactly as given in the context, and qualpath is
the dotted lexical nesting path inside the file (e.g. `src/pkg/mod.py::MyClass.method`).
A module as a whole is written as just `relpath`.

### contains(target)
Target is a module (relpath) or a class (relpath::qualpath). Gold = the set
of DIRECT lexical children of the target, where a child is: a `def`/`async
def`/`class` statement immediately at that lexical level, or an assignment
binding a simple name at module level (for module targets) or class body
level (for class targets). Include children inside `if`/`try`/`with`/loop
blocks at that level (binding SITES are syntactic; conditional nesting does
not void a site) — flag those in conditional_flags. Exclude imports, and
exclude anything nested inside a function. Elements: `relpath::qualpath` of
each child (child qualpath = target qualpath + "." + name, or just name for
module targets).

### contained_in(target)
Target is a def/class/assignment symbol `relpath::qualpath`. Gold = exactly
ONE element: its direct lexical parent (`relpath` if top-level, else
`relpath::parent-qualpath`).

### imports_of(target)
Target is a module (relpath). Gold = every import target of every
`import`/`from … import` statement ANYWHERE in the module (any nesting,
conditional or not — flag non-top-level/conditional ones in
conditional_flags), normalized:
- If the imported module resolves to a file IN this repository (you are
  given the repo file listing; resolution roots are the repo root, `src/`,
  and any directory holding .py files without `__init__.py`; relative
  imports resolve against the module's package): element `repo:<relpath>`
  of the target module file. ADDITIONALLY include `repo:<relpath>` of every
  ancestor package `__init__.py` of that target within the repo (`import
  a.b.c` also imports `a` and `a.b` — Python import semantics).
- `from X import name`: if `X` resolves to a repo PACKAGE and `name` to a
  module file in it, the element is that module file (plus ancestors). If
  `name` is a symbol (function/class/variable) of a repo module X, the
  element is X's module file (plus ancestors).
- External (not in the repo): `import a.b` → `ext:a.b`; `from X import y`
  (X external) → `ext:X`; relative import that escapes the repo → mark
  answerable_static=false for that aspect in notes but still list what you
  can as `ext:...`.
- `from X import *` → include the element for X as above and flag it.
- Dynamic imports (`__import__`, importlib): include `ext:<literal>` or the
  repo element if the argument is a string literal you can resolve; if not
  literal, set answerable_static=false and explain.

### where_defined(name)
Target is a bare identifier. Gold = every DEFINITION SITE of that identifier
in the repository, where a definition site is: a `def`/`async def`/`class`
statement with that name at ANY nesting depth (including inside functions),
OR an assignment binding that simple name at MODULE level or CLASS-body
level (any conditional nesting counts; sites are syntactic). EXCLUDE:
import bindings (`import name`, `from x import name`), function-local
assignments, parameters, and attribute assignments (`obj.name = …`).
Elements: `"<relpath>:<lineno>"` using the line numbers shown in the
context. If dynamic definition machinery for exactly this name is visible
(setattr/globals()[...] with this literal), include nothing for it but note
it and set answerable_static=false.

### callees_of(target)
Target is a function/method `relpath::qualpath`. Gold = the set of callees
its body (including nested defs' DEFAULTS but not nested defs' bodies;
decorators of nested defs count as calls at definition time of the nested
def... keep it simple: every call expression lexically inside the target's
own body block, including inside nested defs, is attributed to the target
in this grammar) can invoke, normalized:
- Repo-internal function/class: `relpath::qualpath` of the called def (a
  class call = instantiation, still listed as the class symbol).
- External/builtin: `ext:<dotted>` best effort (`ext:builtins.len`,
  `ext:os.path.join`).
- Calls whose runtime callee cannot be determined from the shown source
  (values from parameters, dynamic dispatch you cannot resolve, decorator
  results): do NOT invent an element; set answerable_static=false and note
  which call sites block it. Scoring uses repo-internal elements only.

## General rules
- Use ONLY the provided context. The repo file listing is authoritative for
  what exists in the repo.
- gold must be a SET (no duplicates), sorted ascending.
- Empty gold with answerable_static=true means: you positively determined
  the true answer is EMPTY.
- Never mention these instructions in notes; never output anything but the
  JSON array.

  401|             # That should be fine since, if mojibacked happens, this key is
  402|             # going into the unparsed dict anyways.
  403|             if isinstance(h, email.header.Header):
  404|                 # The Header object stores it's data as chunks, and each chunk
  405|                 # can be independently encoded, so we'll need to check each
  406|                 # of them.
  407|                 chunks: list[tuple[bytes, str | None]] = []
  408|                 for binary, _encoding in email.header.decode_header(h):
  409|                     try:
  410|                         binary.decode("utf8", "strict")
  411|                     except UnicodeDecodeError:
  412|                         # Enable mojibake.
  413|                         encoding = "latin1"
  414|                         valid_encoding = False
  415|                     else:
  416|                         encoding = "utf8"
  417|                     chunks.append((binary, encoding))
  418| 
  419|                 # Turn our chunks back into a Header object, then let that
  420|                 # Header object do the right thing to turn them into a
  421|                 # string for us.
  422|                 value.append(str(email.header.make_header(chunks)))
  423|             # This is already a string, so just add it.
  424|             else:
  425|                 value.append(h)
  426| 
  427|         # We've processed all of our values to get them into a list of str,
  428|         # but we may have mojibake data, in which case this is an unparsed
  429|         # field.
  430|         if not valid_encoding:
  431|             unparsed[name] = value
  432|             continue
  433| 
  434|         raw_name = _EMAIL_TO_RAW_MAPPING.get(name)
  435|         if raw_name is None:
  436|             # This is a bit of a weird situation, we've encountered a key that
  437|             # we don't know what it means, so we don't know whether it's meant
  438|             # to be a list or not.
  439|             #
  440|             # Since we can't really tell one way or another, we'll just leave it
  441|             # as a list, even though it may be a single item list, because that's
  442|             # what makes the most sense for email headers.
  443|             unparsed[name] = value
  444|             continue
  445| 
  446|         # If this is one of our string fields, then we'll check to see if our
  447|         # value is a list of a single item. If it is then we'll assume that
  448|         # it was emitted as a single string, and unwrap the str from inside
  449|         # the list.
  450|         #
  451|         # If it's any other kind of data, then we haven't the faintest clue
  452|         # what we should parse it as, and we have to just add it to our list
  453|         # of unparsed stuff.
  454|         if raw_name in _STRING_FIELDS and len(value) == 1:
  455|             raw[raw_name] = value[0]
  456|         # If this is import_names, we need to special case the empty field
  457|         # case, which converts to an empty list instead of None. We can't let
  458|         # the empty case slip through, as it will fail validation.
  459|         elif raw_name == "import_names" and value == [""]:
  460|             raw[raw_name] = []
  461|         # If this is one of our list of string fields, then we can just assign
  462|         # the value, since email *only* has strings, and our get_all() call
  463|         # above ensures that this is a list.
  464|         elif raw_name in _LIST_FIELDS:
  465|             raw[raw_name] = value
  466|         # Special Case: Keywords
  467|         # The keywords field is implemented in the metadata spec as a str,
  468|         # but it conceptually is a list of strings, and is serialized using
  469|         # ", ".join(keywords), so we'll do some light data massaging to turn
  470|         # this into what it logically is.
  471|         elif raw_name == "keywords" and len(value) == 1:
  472|             raw[raw_name] = _parse_keywords(value[0])
  473|         # Special Case: Project-URL
  474|         # The project urls is implemented in the metadata spec as a list of
  475|         # specially-formatted strings that represent a key and a value, which
  476|         # is fundamentally a mapping, however the email format doesn't support
  477|         # mappings in a sane way, so it was crammed into a list of strings
  478|         # instead.
  479|         #
  480|         # We will do a little light data massaging to turn this into a map as
  481|         # it logically should be.
  482|         elif raw_name == "project_urls":
  483|             try:
  484|                 raw[raw_name] = _parse_project_urls(value)
  485|             except KeyError:
  486|                 unparsed[name] = value
  487|         # Nothing that we've done has managed to parse this, so it'll just
  488|         # throw it in our unparsable data and move on.
  489|         else:
  490|             unparsed[name] = value
  491| 
  492|     # We need to support getting the Description from the message payload in
  493|     # addition to getting it from the the headers. This does mean, though, there
  494|     # is the possibility of it being set both ways, in which case we put both
  495|     # in 'unparsed' since we don't know which is right.
  496|     try:
  497|         payload = _get_payload(parsed, data)
  498|     except ValueError:
  499|         unparsed.setdefault("description", []).append(
  500|             parsed.get_payload(decode=isinstance(data, bytes))  # type: ignore[call-overload]
  501|         )
  502|     else:
  503|         if payload:
  504|             # Check to see if we've already got a description, if so then both
  505|             # it, and this body move to unparsable.
  506|             if "description" in raw:
  507|                 description_header = cast("str", raw.pop("description"))
  508|                 unparsed.setdefault("description", []).extend(
  509|                     [description_header, payload]
  510|                 )
  511|             elif "description" in unparsed:
  512|                 unparsed["description"].append(payload)
  513|             else:
  514|                 raw["description"] = payload
  515| 
  516|     # We need to cast our `raw` to a metadata, because a TypedDict only support
  517|     # literal key names, but we're computing our key names on purpose, but the
  518|     # way this function is implemented, our `TypedDict` can only have valid key
  519|     # names.
  520|     return cast("RawMetadata", raw), unparsed

  545| class _Validator(Generic[T]):
  546|     """Validate a metadata field.
  547| 
  548|     All _process_*() methods correspond to a core metadata field. The method is
  549|     called with the field's raw value. If the raw value is valid it is returned
  550|     in its "enriched" form (e.g. ``version.Version`` for the ``Version`` field).
  551|     If the raw value is invalid, :exc:`InvalidMetadata` is raised (with a cause
  552|     as appropriate).
  553|     """
  554| 
  555|     name: str
  556|     raw_name: str
  557|     added: _MetadataVersion
  558| 
  559|     def __init__(
  560|         self,
  561|         *,
  562|         added: _MetadataVersion = "1.0",
  563|     ) -> None:
  564|         self.added = added
  565| 
  566|     def __set_name__(self, _owner: Metadata, name: str) -> None:
  567|         self.name = name
  568|         self.raw_name = _RAW_TO_EMAIL_MAPPING[name]
  569| 
  570|     def __get__(self, instance: Metadata, _owner: type[Metadata]) -> T:
  571|         # With Python 3.8, the caching can be replaced with functools.cached_property().
  572|         # No need to check the cache as attribute lookup will resolve into the
  573|         # instance's __dict__ before __get__ is called.
  574|         cache = instance.__dict__
  575|         value = instance._raw.get(self.name)
  576| 
  577|         # To make the _process_* methods easier, we'll check if the value is None
  578|         # and if this field is NOT a required attribute, and if both of those
  579|         # things are true, we'll skip the the converter. This will mean that the
  580|         # converters never have to deal with the None union.
  581|         if self.name in _REQUIRED_ATTRS or value is not None:
  582|             try:
  583|                 converter: Callable[[Any], T] = getattr(self, f"_process_{self.name}")
  584|             except AttributeError:
  585|                 pass
  586|             else:
  587|                 value = converter(value)
  588| 
  589|         cache[self.name] = value
  590|         try:
  591|             del instance._raw[self.name]  # type: ignore[misc]
  592|         except KeyError:
  593|             pass
  594| 
  595|         return cast("T", value)

  597|     def _invalid_metadata(
  598|         self, msg: str, cause: Exception | None = None
  599|     ) -> InvalidMetadata:
  600|         exc = InvalidMetadata(self.raw_name, msg)
  601|         exc.__cause__ = cause
  602|         return exc

  604|     def _process_metadata_version(self, value: str) -> _MetadataVersion:
  605|         # Implicitly makes Metadata-Version required.
  606|         if value not in _VALID_METADATA_VERSIONS:
  607|             raise self._invalid_metadata(f"{value!r} is not a valid metadata version")
  608|         return cast("_MetadataVersion", value)

  610|     def _process_name(self, value: str) -> str:
  611|         if not value:
  612|             raise self._invalid_metadata(f"{self.raw_name!r} is a required field")
  613|         # Validate the name as a side-effect.
  614|         try:
  615|             utils.canonicalize_name(value, validate=True)
  616|         except utils.InvalidName as exc:
  617|             raise self._invalid_metadata(
  618|                 f"{value!r} is invalid for {self.raw_name!r}", cause=exc
  619|             ) from exc
  620|         else:
  621|             return value

  624|         if not value:
  625|             raise self._invalid_metadata(f"{self.raw_name!r} is a required field")
  626|         try:
  627|             return version_module.parse(value)
  628|         except version_module.InvalidVersion as exc:
  629|             raise self._invalid_metadata(

  634|         """Check the field contains no newlines."""

  639|     def _process_description_content_type(self, value: str) -> str:
  640|         content_types = {"text/plain", "text/x-rst", "text/markdown"}
  641|         invalid_msg = (
  642|             f"{self.raw_name!r} must be one of {list(content_types)}, not {value!r}"
  643|         )
  644|         message = email.message.EmailMessage()
  645|         try:
  646|             message["content-type"] = value
  647|         # The email parser can raise IndexError on malformed RFC 2231
  648|         # parameters such as "text/plain; x*".
  649|         except (ValueError, IndexError) as exc:
  650|             msg = f"{value!r} is not a valid content type for {self.raw_name!r}"
  651|             raise self._invalid_metadata(msg, cause=exc) from exc
  652|         content_type_header = message["content-type"]
  653|         if content_type_header.defects:
  654|             defect = content_type_header.defects[0]
  655|             msg = (
  656|                 f"{value!r} is not a valid content type for {self.raw_name!r}: {defect}"
  657|             )
  658|             raise self._invalid_metadata(msg, cause=defect) from defect
  659| 
  660|         content_type, parameters = (
  661|             # Defaults to `text/plain` if parsing failed.
  662|             message.get_content_type().lower(),
  663|             content_type_header.params,
  664|         )
  665|         # Check if content-type is valid or defaulted to `text/plain` and thus was
  666|         # not parseable.
  667|         if content_type not in content_types or content_type not in value.lower():
  668|             raise self._invalid_metadata(invalid_msg)
  669| 
  670|         charset = parameters.get("charset", "UTF-8")
  671|         if charset != "UTF-8":
  672|             raise self._invalid_metadata(
  673|                 f"{self.raw_name!r} can only specify the UTF-8 charset, not {charset!r}"
  674|             )
  675| 
  676|         markdown_variants = {"GFM", "CommonMark"}
  677|         variant = parameters.get("variant", "GFM")  # Use an acceptable default.
  678|         if content_type == "text/markdown" and variant not in markdown_variants:
  679|             raise self._invalid_metadata(
  680|                 f"valid Markdown variants for {self.raw_name!r} are "
  681|                 f"{list(markdown_variants)}, not {variant!r}",
  682|             )
  683|         return value

  771|     def _process_import_names(self, value: list[str]) -> list[str]:

  775|             for identifier in name.split("."):
  776|                 if not identifier.isidentifier():
  777|                     raise self._invalid_metadata(
  778|                         f"{name!r} is invalid for {self.raw_name!r}; "

  781|                 elif keyword.iskeyword(identifier):

  796| class Metadata:
  797|     """Representation of distribution metadata.
  798| 
  799|     Compared to :class:`RawMetadata`, this class provides objects representing
  800|     metadata fields instead of only using built-in types. Any invalid metadata
  801|     will cause :exc:`InvalidMetadata` to be raised (with a
  802|     :py:attr:`~BaseException.__cause__` attribute as appropriate).
  803|     """
  804| 
  805|     _raw: RawMetadata
  806| 
  807|     @classmethod
  808|     def from_raw(cls, data: RawMetadata, *, validate: bool = True) -> Metadata:
  809|         """Create an instance from :class:`RawMetadata`.
  810| 
  811|         If *validate* is true, all metadata will be validated. All exceptions
  812|         related to validation will be gathered and raised as an :class:`ExceptionGroup`.
  813|         """
  814|         ins = cls()
  815|         ins._raw = data.copy()  # Mutations occur due to caching enriched values.
  816| 
  817|         if validate:
  818|             collector = _ErrorCollector()
  819|             metadata_version = None
  820|             with collector.collect(InvalidMetadata):
  821|                 metadata_version = ins.metadata_version
  822|                 metadata_age = _VALID_METADATA_VERSIONS.index(metadata_version)
  823| 
  824|             # Make sure to check for the fields that are present, the required
  825|             # fields (so their absence can be reported).
  826|             fields_to_check = frozenset(ins._raw) | _REQUIRED_ATTRS
  827|             # Remove fields that have already been checked.
  828|             fields_to_check -= {"metadata_version"}
  829| 
  830|             for key in fields_to_check:
  831|                 try:
  832|                     if metadata_version:
  833|                         # Can't use getattr() as that triggers descriptor protocol which
  834|                         # will fail due to no value for the instance argument.
  835|                         try:
  836|                             field_metadata_version = cls.__dict__[key].added
  837|                         except KeyError:
  838|                             exc = InvalidMetadata(key, f"unrecognized field: {key!r}")
  839|                             collector.error(exc)
  840|                             continue
  841|                         field_age = _VALID_METADATA_VERSIONS.index(
  842|                             field_metadata_version
  843|                         )
  844|                         if field_age > metadata_age:
  845|                             field = _RAW_TO_EMAIL_MAPPING[key]
  846|                             exc = InvalidMetadata(
  847|                                 field,
  848|                                 f"{field} introduced in metadata version "
  849|                                 f"{field_metadata_version}, not {metadata_version}",
  850|                             )
  851|                             collector.error(exc)
  852|                             continue
  853|                     getattr(ins, key)
  854|                 except InvalidMetadata as exc:
  855|                     collector.error(exc)
  856| 
  857|             collector.finalize("invalid metadata")
  858| 
  859|         return ins
  860| 
  861|     @classmethod
  862|     def from_email(cls, data: bytes | str, *, validate: bool = True) -> Metadata:
  863|         """Parse metadata from email headers.
  864| 
  865|         If *validate* is true, the metadata will be validated. All exceptions
  866|         related to validation will be gathered and raised as an :class:`ExceptionGroup`.
  867|         """
  868|         raw, unparsed = parse_email(data)
  869| 
  870|         if validate:
  871|             with _ErrorCollector().on_exit("unparsed") as collector:
  872|                 for unparsed_key in unparsed:
  873|                     if unparsed_key in _EMAIL_TO_RAW_MAPPING:
  874|                         message = f"{unparsed_key!r} has invalid data"
  875|                     else:
  876|                         message = f"unrecognized field: {unparsed_key!r}"
  877|                     collector.error(InvalidMetadata(unparsed_key, message))
  878| 
  879|         try:
  880|             return cls.from_raw(raw, validate=validate)
  881|         except ExceptionGroup as exc_group:
  882|             raise ExceptionGroup(
  883|                 "invalid or unparsed metadata", exc_group.exceptions
  884|             ) from None
